# Операції з LLM у Lumi: маршрутизація моделей і зниження вартості

Зараз **усі** виклики моделі йдуть на `claude-opus-4-8`. Цей звіт перелічує **кожну** операцію з LLM
(розділ 1), пропонує дешевшу модель + thinking on/off для кожної операції **й кожного інструмента**
(розділ 2), зводить **увесь аналіз вартості в один розділ 3** (операції + інструменти + сценарії), і
закриває ризиками (4) та рекомендацією (5). Усі цифри — **виміряні** з `.lumi/cache-log.jsonl` (2 738
викликів) та `.lumi/tool-log.jsonl` (822 виконання тулів) за 16–22 червня.

---

## 0. Контекст і передумови

- **Зараз:** одна модель на все — єдиний інжектований `LLMClient` (`self._llm`). `LUMI_THINKING=on`,
  `LUMI_EFFORT=medium` діють **лише для головної відповіді**; усі housekeeping-виклики
  (`_housekeeping_reply`) **примусово вимикають thinking**.
- **Ціни Anthropic** (за 1M токенів): **Opus 4.8 — $5 / $25**, **Sonnet 4.6 — $3 / $15**,
  **Haiku 4.5 — $1 / $5**. cache-read = 10% від input; cache-write = ×2 при 1h TTL (у тебе `LUMI_PROMPT_CACHE_TTL=1h`).
  Тобто Sonnet ≈ **0.6×** ціни Opus, Haiku ≈ **0.2×** (множник однаковий на input/output/cache).
- **Передумова до впровадження:** per-operation роутинг **ще не побудований** — щоб застосувати
  рекомендації, треба тримати 2–3 клієнти й маршрутизувати виклики (reply → Opus; `_housekeeping_reply`
  та `think` → дешевший клієнт). Сема `LLMClient` уже інжектована, тож зміна **мала й локальна**.

## 4. Ризики й застереження

- **Per-op роутинг треба спершу побудувати** (≥2 клієнти, маршрутизація `_housekeeping_reply`/`think`/тул-кроків).
  Без цього економія нульова — це інженерна передумова, а не конфіг-флаг.
- **`tool` (21%) — найбільший «другий» важіль, але найтонший.** Виклик тула в reply-петлі = продовження того
  ж туру; даунгрейд б'є по якості відповіді. Дешевше й безпечніше **скоротити кроки** (ліміти `*_MAX_CALLS`)
  та **кепнути розмір результату `read_file`**, ніж знижувати модель.
- **Мікс моделей в одній історії повідомлень = ризик когерентності** (модель бачить чужі `tool_use`/`thinking`-блоки).
  Тому не перемикай «на кожен тул» наосліп — дотримуйся патерну з розділу 5.
- **Мікс операцій плаває.** Цей тиждень — «важкий» на `tool`/`think` (розробка/тести). У «чистий» чат-тиждень
  частка `reply` вища → економія в **$** менша, але **%** схожий.
- Цифри — оцінка зі списку цін, не білінг; `cache-log` міг не покрити 100% викликів.

## 5. Рекомендація

1. **Старт — «збалансований + per-tool» (✅, −26%):** `reply` на **Opus**; `think`/`mood`/`knowledge`/`images`/`recall`/`read_file` → **Sonnet**; housekeeping + `write`/`navigation`-тули → **Haiku**.
2. **Безпечний патерн впровадження тул-петлі** (щоб не мішати моделі наосліп):
   - **Проміжні кроки** (читання/навігація/запис/фетч — користувач їх **не бачить**) → дешева модель.
   - **Фінальний крок, що пише видиму відповідь** → **Opus**.
   - **think-петля цілком** → Sonnet.
   - **Кеп розміру результату `read_file`/`read_around`/`search_files`** — зрізає 29%-ну статтю **без** зміни моделі.
3. **Не чіпати головну відповідь** (`reply`, Opus + thinking) — це продукт.
4. **Виміряти 2 тижні**, тоді вирішувати про агресивніші кроки (`think`→Haiku тощо).
5. **Бонус-важелі поза моделлю:** менше тул-кроків, `effort=low` на простих турнах (менше output за $25/1M);
   кешування вже економить ~$232/міс, а v0.36 (факти-ядро) зрізала вхід на турн ще ~17%.

> **TL;DR:** `reply` — 53% вартості й лишається на Opus. Безпечна економія від маршрутизації моделей —
> **−8…−14%**, а зі **збалансованим + per-tool** — **−26% (≈ $63/міс)**, без удару по голосу: Стхіра
> **думає й копається** на дешевших моделях, але **говорить** на Opus.

# Додаток A — альтернатива: моделі OpenAI

> **Дисклеймер для всіх додатків:** ціни **приблизні** (станом на ~2026) — звір з офіційним прайсом.
> Це **виключно вартісний** аналіз; **якість** української персони/голосу тут **не оцінюється**.
> Не-Anthropic моделі **втрачають Anthropic-кеш** (який економить ~$232/міс), тож кеш-важкі операції
> (`reply`/`tool`/`think`) я порахував **за найгіршим випадком** — увесь кешований префікс

**Гіпотетична стеля (усе на одній дешевій моделі, ігноруючи якість):** GPT-4o-mini ≈ **$13/тиж**,
MiniMax ≈ **$18/тиж**, DeepSeek ≈ **$24/тиж** — проти **$208.68** на Opus (**−89…−94%**). Це показує, що

**Критичні застереження (саме для приватного компаньйона):**
- 🔒 **Приватність / юрисдикція.** **DeepSeek і MiniMax — китайські провайдери**: інтимна памʼять і розмови
  Стхіри підуть на їхні сервери. Для приватного компаньйона (де проєкт навмисно тримає **локальні
  ембединги**, приватний store) це **серйозний компроміс**. OpenAI/Anthropic — США (теж не локально, але
  інша юрисдикція). **Єдиний по-справжньому приватний шлях — self-host** (Llama/Qwen на своєму залізі).
- **Якість голосу.** DeepSeek/MiniMax сильні в коді/логіці, але українська лірична персона, «туман»,
  внутрішній монолог — велике питання. Для `reply`/`think` ризик **високий**.
- **Кеш.** Без Anthropic-кешу кеш-важкі операції перевиставляють префікс; власний кеш DeepSeek (cache-hit
  $0.07) це помʼякшує, MiniMax — невідомо.
- **Structured output / тули.** DeepSeek — через OpenAI-сумісний адаптер (вже є). MiniMax — JSON-режим.
  v0.3-гейт валідації ловить розбіжності.

**Рекомендація (DeepSeek/MiniMax/інші):**
- **Housekeeping** (самарі/факти/компакція/дайджести) — найкращий і **єдиний безпечний** кандидат: тексту
  персони там нема, тож DeepSeek/MiniMax/Gemini-Flash дають ≈$1/тиж. Але, як і з OpenAI, це **лише ~$3/тиж
  понад Haiku** — не вартує окремого провайдера саме для цього.
- **`reply`/`think`** — **не рекомендую** через приватність (китайські сервери) + ризик голосу.
- Якщо ціль — **максимум приватності + дешевизна** і ти готовий до інженерії — **self-host Llama/Qwen**
  на housekeeping/think: дані не залишають машину (як локальні ембединги), вартість ≈ електрика.

# Додаток C — Reasoning-тір альтернативи Opus 4.8 (звід)

Окремо — **прямі заміни Opus 4.8 на головній відповіді**: моделі-**reasoning** (з thinking), тож, на відміну
від mini-тірів у Додатках A/B, вони кандидати **і на `reply`**, не лише на housekeeping. Усі три —
**нерелізовані станом на мій cutoff**, тож ціни — **проксі-оцінки** (звірити з офіційним прайсом, коли вийдуть):
GPT-5.5 Thinking ≈ GPT-5; DeepSeek-V4-Pro Thinking ≈ DeepSeek-R1; MiniMax-M3 Thinking ≈ MiniMax M-серія.

### Ціни (проксі, за 1M токенів)

| Модель | input | output | cached input |
|---|---:|---:|---:|
| **Opus 4.8** (зараз) | $5.00 | $25.0 | $0.50 |
| **GPT-5.5 Thinking** | ~$1.25 | ~$10.0 | ~$0.125 |
| **DeepSeek-V4-Pro Thinking** | ~$0.55 | ~$2.19 | ~$0.14 |
| **MiniMax-M3 Thinking** | ~$0.30 | ~$1.65 | ~$0.08 |

### Вартість як заміна Opus 4.8 (тиждень виміру)

| Модель | `reply` | весь стек | Δ vs Opus 4.8 |
|---|---:|---:|---:|
| **Opus 4.8** (зараз) | $110 | **$208** | — |
| **GPT-5.5 Thinking** | $26–63 | **$50–115** | **−45…−76%** |
| **DeepSeek-V4-Pro Thinking** | $13–27 | **$24–48** | **−77…−88%** |
| **MiniMax-M3 Thinking** | $8–15 | **$14–27** | **−87…−93%** |

> Діапазон: нижня межа — **з власним кешем** моделі; верхня — **без кешу** (префікс перевиставляється щотурна).
> Тобто навіть найгірший випадок будь-якої з трьох **дешевший за Opus** на стеку.

### Як вибирати — три осі

| Вісь | **GPT-5.5 Thinking** | **DeepSeek-V4-Pro Thinking** | **MiniMax-M3 Thinking** |
|---|---|---|---|
| **Ціна** | найдорожча з трьох (та все одно −76%) | середня (−88%) | найдешевша (−93%) |
| **Якість / UA-голос** | фронтір, найближча до Opus; треба A/B + перетюнінг | сильний reasoning; голос — питання | найменш перевірена на ліричній персоні |
| **Приватність** | 🇺🇸 США — паритет з Anthropic ✓ | 🔒 КНР — інтимна памʼять на чужих серверах | 🔒 КНР — те саме |

### Рекомендація

- **Найкраща заміна Opus 4.8 = `GPT-5.5 Thinking`.** Фронтір-reasoning + приватність рівня Anthropic (США) +
  **−76%** на стеку (`reply` $110 → $26). Єдина з трьох, кого варто пускати на **`reply`**.
- **`DeepSeek-V4-Pro` / `MiniMax-M3 Thinking`** — **дешевші** ($14–48/тиж), але **китайські сервери** роблять
  їх **неприйнятними для `reply`/`think`** приватного компаньйона (інтимна памʼять їде на чужий бекенд).
  Прийнятні лише як **open-weight self-host** (тоді дані не залишають машину).
- **Спосіб упровадження (для будь-якої):** пілот на `reply` з `effort=medium`, **A/B проти Opus 4.8 на голосі
  й глибині 1–2 тижні**; рішення **за якістю, не ціною** — канон Стхіри тюнено під Claude, тож потрібен
  перетюнінг тону / «туману» / внутрішнього монологу.
