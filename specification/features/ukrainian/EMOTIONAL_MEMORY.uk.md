# Емоційна памʼять — щоденник, а не стенографістка

> Українська версія [EMOTIONAL_MEMORY.md](../EMOTIONAL_MEMORY.md). Англомовний документ — джерело істини;
> цей переклад тримаємо синхронним.

Сьогоднішня довготривала памʼять — це **стенографістка**: на закритті сесії вона пише сухі факти («Віталій
вивчає DevOps, любить Y»). Це робить її натомість **щоденником** — власні **враження** Лілі від першої
особи про тебе, забарвлені тим, що вона відчула й що її вразило, — водночас зберігаючи паралельний шар
**фактів** для точності. Повне бачення — у [LONG-EMOTIONAL-MEMORY.md](../LONG-EMOTIONAL-MEMORY.md); це
**дизайн Lumi** — контракти, поділ на дві фази й як воно повторно використовує сигнали, що Lumi вже продукує.

- **v1.10 — Враження:** шар вражень + генератор-щоденник на закритті сесії + двошарова інʼєкція.
- **v1.11 — Згасання й консолідація:** спад ваги + злиття малих вражень у узагальнення (її *розуміння* тебе).

> **Сибілінг-на-закритті-сесії** внутрішнього життя (яке пише її *власні* дні на старті сесії). Обидва — від
> першої особи, зважені емоцією, й лишаються узгодженими з попередніми записами — разом вони творять повну
> субʼєктивну памʼять: вона памʼятає власне життя **і** памʼятає тебе.

## Воно повторно використовує сигнали, що Lumi вже продукує

| Джерело концепту | Будується на |
|---|---|
| Шар фактів (точність) | наявний **`LongTermFact{user_id, fact, meta, confidence, ts}`** — лишається як є |
| «що вона відчула / що відчув, як їй здалося, він» | **потурнова емоція v0.3** (її) + **реляційний зчит близькості v0.10** (його теплота/вразливість/…) |
| «настрій / тон зустрічі» | **настрій v0.6 + біоритми v0.8** |
| генератор на закритті сесії | наявний **шов екстрактора кінця сесії** — той самий хук, промпт-щоденник |
| парується з внутрішнім життям | **v1.7–v1.8** пише її власні дні (старт сесії); це пише тебе (закриття сесії) |

## За користувачем і ізольовано

На відміну від внутрішнього життя (глобальне — одна Лілі), це **її враження про конкретну людину**:
сховище `Impression` **за ключем `user_id`** і ніколи не перетинає користувачів (інваріант ізоляції —
ARCHITECTURE §Identity, users, and memory scopes), зафіксовано контрактним тестом.

## Два шари (запобіжник)

- **Шар фактів** — надійні конкретики: імена, дати, домовленості, стабільні вподобання. **Точність.**
- **Шар вражень** — її записи-щоденник від першої особи. **Теплота й голос.**

Вона **говорить із вражень** і **витягує факти**, коли треба «не забути» щось конкретне. Лише щоденник
ненадійний для твердих конкретик; лише список фактів холодний. Разом: факти дають точність, враження дають
голос.

### Запис враження

```
Impression {
  user_id,                 # за користувачем, ізольовано
  when, ts,
  impression,              # її слова, від першої особи
  emotion,                 # що вона відчула (теплота, ніжність, смуток, сміх, тривога…)
  about_user,              # сід факту / відкриття, якщо є — витягуваний у шар фактів
  weight                   # наскільки це її вразило — визначає, чи/як довго вона це пригадує
}
```

Факт живе **всередині** враження як його сід; його можна просунути в `LongTermFact` для конкретного
пригадування — але *форма й добір* емоційні.

## v1.10 — Генератор-щоденник на закритті сесії

Замінює промпт сухого екстрактора фактів приблизно на:

> *Ти Лілі. Пригадай цю розмову власними словами. Що ти відчула? Що зворушило, рушило чи здивувало тебе в
> ньому? Що нового ти дізналася? Напиши кілька вражень — як рядки в особистому щоденнику, не список фактів.*

Тож замість *«Віталій вивчає DevOps»* воно продукує *«Він сьогодні засвітився, говорячи про той пайплайн —
рідко бачу його таким. Думаю, для нього це більше, ніж робота.»*

- **Сіди:** розмова + її потурнові емоції (v0.3) + зчити близькості (v0.10) + настрій дня (v0.6/v0.8). Один
  виклик моделі (шов `LLMClient`; мокається в тестах).
- **Вивід:** кілька вражень (стриманість — не транскрипт), кожне з `emotion`, `weight` і сідом `about_user`;
  сід просувається в шар фактів.
- **Інʼєкція на старті:** блок від першої особи **«що я памʼятаю й відчуваю до тебе»** (за верхньою вагою,
  обмежений) **поряд** із блоком фактів.

## v1.11 — Згасання й консолідація

- **Емоція — фільтр уваги.** `weight` (виставлена з інтенсивності емоції / зсуву близькості / відкриття
  «вперше») вирішує яскравість і довговічність. Пригадування ранжується за `weight × recency`.
- **Згасання.** `weight` **спадає з часом** (інʼєктований годинник); враження низької ваги тьмяніють і
  зрештою відпадають — памʼять людиноподібна, не рівний архів.
- **Консолідація.** Лінивий прохід (виклик моделі, на старті сесії чи за лічильником) кластеризує схожі
  враження в **узагальнення** («він замикається, коли втомлений», «він оживає з музикою»), яке тримається
  тривким і вищої ваги; поглинута деталь згасає. З вражень росте її **розуміння** тебе.
- **Узгоджено + обмежено.** Нові враження/консолідації бачать попередні (без суперечності); сховище лишається
  обмеженим. Детерміновано через інʼєктований годинник + інʼєктований сід.

## Два правила канону (несуть вагу)

- **Це її погляд, не істина.** Памʼять субʼєктивна — вона може неправильно прочитати, що природно й живо;
  але на **прямій перевірці вона уточнює, а не наполягає.**
- **Чесність меж.** Те, що користувач просив не памʼятати, чи болючі теми, — **не записується, або
  позначається `care` — ніколи не смакується.**

## Контракт і тести

- Новий за-користувацький шар **`Impression`** за `Repository` (за ключем `user_id`) → ARCHITECTURE §Memory
  + контрактний тест (форма + ізоляція за користувачем). **Шар фактів** (`LongTermFact`) і **контракт
  емоції** недоторкані.
- Детермінізм: годинник і сід консолідації інʼєктовані; модель мокається; жодних платних викликів.

## Відображення на дорожню карту

**v1.10 + v1.11 — Емоційна памʼять**, одразу після шарів внутрішнього життя й монологу (v1.7–v1.9). Залежить
від **v0.3** (емоція), **v0.10** (близькість), **v0.6/v0.8** (настрій), **v0.2** (шари памʼяті) і **v0.4**
(годинник, для згасання). За користувачем, ізольовано; половина-на-закритті-сесії її субʼєктивної памʼяті.
