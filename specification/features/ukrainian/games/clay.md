# Clay

> Світ, який ліпиш блок за блоком. Земляний брат Silt — воксельний субстрат, у якому живе істота.

Стартове ядро: бот, що живе у воксельному світі на швидких тіках, і окремий
мозок-заглушка, який на запит каже, що робити далі. Запит передає **потреби**
і **реальну картину навколо**. Мозок міняється на Лілі (Lumi) без жодних змін у
тілі.

## Головний принцип

Тіло живе на швидких тіках саме по собі. Мозок консультують лише коли щось
важить. Між зверненнями до мозку бот рухається, його потреби спадають, він
доробляє останній намір — усе це чистим кодом. Саме розрив між думками створює
відчуття безперервного життя, а не покрокового ходу.

```
тіло (швидкі тіки)  ──perception──>  мозок  ──intent──>  тіло
   рух, pathfinding,                 "що далі?"          виконує намір
   спад потреб,                                          сотні тіків сам
   виконання наміру
```

- **Perception** (тіло -> мозок): потреби (рівні + терміновість) + локальний
  патч світу навколо + найближча незвідана клітина + місце для будівництва.
- **Intent** (мозок -> тіло): один високорівневий намір (`goto` / `build` /
  `rest` / `wander` / `idle`) з причиною. Тіло виконує його без нових запитів,
  поки не завершить.

Мозок будиться лише коли: холодний старт, суттєвий намір завершено, потреба
перетнула поріг під час дрейфу, або спрацював heartbeat. Інакше тіло дрейфує
саме, без запиту. У headless-прогоні це ~1 звернення на 18 тіків.

## Потреби

Три загальні драйви-заглушки. Форма та сама, що в Лілі — підставляться її шість
NEEDS:

| тут (заглушка) | Лілі | заземлення у світі |
|---|---|---|
| novelty | новизна | відвідування нових клітин рельєфу |
| creation | творчість | ставлення блоків |
| rest | (тілесна життєва сила) | відпочинок |

У Лілі додаються усамітнення, зв'язок, свобода, сенс — їх заземлює багатший світ.

## Структура

```
sim/
  world.py        воксельний світ: рельєф, блоки, "картина навколо"
  needs.py        драйви: спад, поповнення, терміновість
  intent.py       намір (мозок -> тіло)
  perception.py   знімок потреб + оточення (тіло -> мозок); to_prompt_dict() = шов для LLM
  brain.py        Brain (інтерфейс) + StubBrain (правила). ЦЕ міняється на Лілі
  body.py         життя на тіках: виконання наміру, коли будити мозок, pathfinding
  simulation.py   зв'язує world + body + brain, метод step()
brains/
  llm_brain_template.py   шаблон LLMBrain — як саме підключається Лілі
run_headless.py   прогін у терміналі, працює будь-де (без графіки)
game.py           воксельний рендер на Ursina (запускати локально)
```

## Запуск

Ядро без залежностей:

```
python run_headless.py
python run_headless.py --ticks 4000 --map-every 500
```

Воксельний 3D-вид (локально):

```
pip install ursina
python game.py
```

`@` — бот, `#` — поставлені блоки, `.` — досліджене (у headless-мапі).

## Перемикання на Лілі

Тіло не змінюється. Увесь контракт — `request(perception)` / `poll()` / `busy`.
StubBrain відповідає миттєво; LLMBrain відповідає в окремому потоці, тож тіло
далі виконує попередній намір ті секунди, поки модель думає. Прийшла
відповідь — `poll()` віддає її, тіло підхоплює на наступному тіку. Жодного
фризу, жодного переписування.

```
pip install anthropic
export ANTHROPIC_API_KEY=...
```

```python
from sim import Simulation, World
from brains.llm_brain_template import LLMBrain
sim = Simulation(world=World(), brain=LLMBrain())   # замість StubBrain
```

Щоб зробити з нього Лілі: підставити її канон у SYSTEM_PROMPT, віддати шість
NEEDS замість трьох, додати шари особистості (OPINIONS / SELF_REGARD /
RELATIONAL_FEELINGS / INNER_MONOLOGUE) у контекст, і за бажанням додати поле
`say` поруч із наміром. Контракт Intent лишається той самий.

## Важелі налаштування

| важіль | де | ефект |
|---|---|---|
| `TICK_INTERVAL` | game.py | швидкість життя |
| `HEARTBEAT` | body.py | максимальна пауза між запитами до мозку |
| `decay` потреб | needs.py | як швидко наростають потреби |
| `ACT_THRESHOLD` | brain.py | наскільки терміновою має стати потреба, щоб діяти |

Гранулярність наміру vs. частота запитів: дрібні `goto` по одній клітині дають
багато запитів; для LLM варто видавати ширші наміри ("досліди той регіон"), щоб
один запит покривав сотні тіків.

## Природні наступні кроки

- A* + вартість стрибка замість жадібного кроку (нерівний рельєф).
- Серіалізація стану на диск (продовження після перезапуску, як снапшоти Silt).
- Ширші наміри для LLM (менше запитів, довші дуги поведінки).
- Поле `say` / inner monologue у наміри під голос Лілі.
