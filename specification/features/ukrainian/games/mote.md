# mote

Окремий самостійний проєкт: кольоровий ASCII-світ у терміналі, у якому сама по
собі живе маленька істота. Вона досліджує, будує й відпочиває, керована
потребами, а її розум — модель Anthropic (Haiku за замовчуванням, Opus
опційно), яку час від часу запитують «що далі». Її внутрішній голос видно в
журналі думок. Без API-ключа вмикається тихий розум на правилах, тож світ
працює завжди.

Це не частина Clay — своя кодова база, свій рушій, свій репозиторій. Спільна
лише ідея: тіло на швидких тіках і окремий розум за чітким інтерфейсом.

## Як це влаштовано

Тіло живе на швидких тіках саме по собі — рухається, його потреби спадають, воно
виконує останній намір. Розум консультують лише коли щось важить, і відповідь
приходить в окремому потоці, тож тіло не завмирає ті секунди, поки модель думає.

```
тіло (швидкі тіки)  ──perception──>  розум (LLM)  ──intent──>  тіло
   спад потреб, рух,                  "що далі?"               виконує намір
   виконання наміру,                  + say (голос)            багато тіків сам
   дрейф коли вільне
```

- **Perception** (тіло -> розум): потреби (рівні + терміновість) і світ навколо
  (локальний патч, кілька незвіданих варіантів, місце під забудову).
- **Intent** (розум -> тіло): один намір (`goto` / `build` / `rest` / `wander`)
  плюс `say` — короткий рядок внутрішнього голосу для журналу думок.

Розум будиться лише коли: холодний старт, суттєвий намір завершено, потреба
перетнула поріг під час дрейфу, або heartbeat. Між цим тіло дрейфує саме.

## Розум (LLM)

За замовчуванням `LLMBrain` на **Haiku** (`claude-haiku-4-5-20251001`); клавішею
`o` або прапорцем `--opus` перемикається на **Opus**. Він:

- працює в окремому потоці (`request` / `poll` / `busy`), тіло не блокується;
- сам обмежує частоту викликів (`min_call_interval`, типово 4 с), щоб
  безперервне життя не засипало API запитами — між викликами тіло просто дрейфує;
- на будь-яку помилку чи відсутність ключа тихо відступає до наміру-дрейфу.

**Вартість.** Кожне рішення — один виклик API. Реальна частота — приблизно один
виклик на кілька секунд (рідше, коли істота спокійна). Важелі: `min_call_interval`
у `brain.py`, `HEARTBEAT` у `body.py`, `--interval` (швидкість тіків).

Якщо `ANTHROPIC_API_KEY` не заданий — `make_brain` повертає `RuleBrain` (миттєвий,
безкоштовний), і світ усе одно живе.

## Потреби

| потреба | заземлення |
|---|---|
| novelty | досягнення незвіданих клітин |
| creation | ставлення блоку |
| rest | спокій на місці |

## Структура

```
mote/
  world.py        рельєф, блоки, "картина навколо", незвідані варіанти
  needs.py        драйви: спад, поповнення, терміновість
  intent.py       намір + say (внутрішній голос)
  perception.py   знімок потреб + оточення; to_prompt_dict() -> промпт
  brain.py        Brain (інтерфейс), LLMBrain (Haiku/Opus, потоковий), RuleBrain
  body.py         життя на тіках, виконання наміру, коли будити розум
  __init__.py     Simulation + експорти
run.py            curses-застосунок: кольоровий ASCII-світ + журнал думок
run_headless.py   те саме без curses (лог рішень і думок, для дебагу)
requirements.txt
```

## Запуск

```
pip install anthropic
export ANTHROPIC_API_KEY=...
python run.py
```

Варіанти:

```
python run.py --opus              # Opus замість Haiku
python run.py --interval 0.06     # швидше
python run.py --width 64 --depth 30
python run.py --rule              # примусово безкоштовний rule-розум
python run_headless.py --rule     # дебаг без графіки
```

Керування: `q` вихід, `space` пауза, `+/-` швидкість, `o` Haiku/Opus.

`@` — істота, `#` — поставлені блоки, відтінки рельєфу — висота, яскравіші
клітини — вже відвідане. Праворуч: стан розуму, потреби, поточна дія, журнал думок.

## Зробити з цього персонажа

Розум — єдиний шов. Підстав свій канон у `SYSTEM_PROMPT` (`brain.py`), розшир
набір потреб, додай контекст особистості — контракт `Intent` (із полем `say`)
лишається той самий, а тіло й рендер не змінюються.

## Природні наступні кроки

- A* із вартістю стрибка замість жадібного кроку (нерівний рельєф).
- Серіалізація стану (продовження після перезапуску).
- Двошаровий розум: Haiku для рутинного «що далі», Opus для рідкісних глибших рішень.
- Памʼять істоти: стислий журнал минулого в перцепцію, щоб голос мав тяглість.
