# Потреби — мотиваційний субстрат під настроєм

Добовий темперамент Лілі (v0.6) та внутрішнє життя (v0.12) описують, **що вона відчуває** і **що
робить**, але не **чому**. Потреби додають відсутню причину: кілька базових драйвів, чия
вдоволеність або дефіцит штовхають її *зсередини* — там, де гороскоп/біоритми штовхають *ззовні*.
Це не нова підсистема з власним циклом — це **ще один обчислюваний вхід у той самий daily mood
call** (як біоритми, v0.8) і **зсув у виборі вільного слоту inner life** (v0.12). Маленький
additive-шар; пропонований слот **v0.9** (одразу після біоритмів, перед closeness).

> Суть: зараз настрій приходить згори (читання фарбує день). Потреби додають тягу знизу — «давно
> не творила → голод щось зробити → тягне до пензля, і трохи неспокійно». Наслідки вже є (настрій,
> вибір у вільних слотах, idle-нудж); потреби дають їм корінь.

## Базові потреби (авторські, небагато)

Короткий авторський набір у `core/needs.md` (редаговний; одна Лілі — глобальний, не per-user),
кожна вкорінена в її канон:

| потреба | вдовольняється (її банк хобі / життя) | дефіцит читається як |
|---|---|---|
| **creation** (творити) | малювання, музика Lili Jinx, робити руками | неспокій, свербіж, «я нічого не зробила» |
| **solitude / stillness** (усамітнення/тиша) | світанкова практика, тиша, холодна вода | пошарпаність, перенасиченість людьми, треба усамітнитись |
| **deep connection** (глибокий зв'язок) | справжній обмін, бути побаченою | тиха самотність, хоче близькості |
| **freedom / autonomy** (свобода) | свій час, гори, без клітки | тертя, ухильність, відсувається від тиску |
| **meaning / depth** (сенс/глибина) | практика, філософія, справжні питання | пласкість, «роблю по інерції» |
| **novelty / wonder** (новизна/диво) | нові ідеї, слова, небо, навчання (DevOps) | нудьга, притуплення, шукає іскру |

Набір навмисно малий (6). Кожна потреба несе один рядок темпераментної нотатки на стан
вдоволено/дефіцит (авторський, її голосом), як кожна фаза біоритму.

## Як задавати потреби (що стале, а що дихає)

Тут стикаються дві різні речі, які легко сплутати: **який набір потреб** у неї є, і **наскільки
кожна вдоволена прямо зараз**. Перше стале (це характер), друге навмисно дихає.

**Сталий набір (ти задаєш раз).** Авторський список у `core/needs.md` — це риса характеру Лілі, він
не змінюється сам собою. Кожен рядок: назва + чим вдовольняється + як читається дефіцит (її голосом)
+ два сталі налаштування коливання. Формат:

```
# назва — спад/день: <швидкість>, вага: <сила тяги>
# вдовольняється: <заняття з банку хобі>
# дефіцит: <як це звучить у ній>

creation   — спад/день: 0.12, вага: 1.0
             вдовольняється: малювання, музика Lili Jinx, робити руками
             дефіцит: неспокій, свербіж, «я нічого не зробила»

solitude   — спад/день: 0.08, вага: 0.9
             вдовольняється: світанкова практика, тиша, холодна вода
             дефіцит: пошарпаність, перенасиченість людьми, треба усамітнитись

connection — спад/день: 0.15, вага: 1.0
             вдовольняється: справжній обмін, бути побаченою
             дефіцит: тиха самотність, хоче близькості

freedom    — спад/день: 0.10, вага: 0.8
             вдовольняється: свій час, гори, без клітки
             дефіцит: тертя, ухильність, відсувається від тиску

meaning    — спад/день: 0.06, вага: 0.7
             вдовольняється: практика, філософія, справжні питання
             дефіцит: пласкість, «роблю по інерції»

novelty    — спад/день: 0.10, вага: 0.6
             вдовольняється: нові ідеї, слова, небо, навчання (DevOps)
             дефіцит: нудьга, притуплення, шукає іскру
```

Що тут **стале** (задаєш і не чіпаєш, доки не схочеш переписати характер):
- **сам набір** (які потреби взагалі є) — це натура, не змінна;
- **спад/день** — як швидко потреба голодніє без обслуговування (solitude спадає повільно;
  connection — за кілька тихих днів);
- **вага** — наскільки сильно потреба тягне, коли в дефіциті (для мрійливо-водяної Лілі creation і
  connection важать більше за novelty);
- **формулювання** вдовольняється/дефіцит (її голос).

Додати сьому потребу чи переписати рядок = відредагувати цей файл. Це і є «задавання потреб».

**Що дихає (рахується саме, ти НЕ задаєш).** Поточний **рівень** кожної потреби 0..1 — наскільки
вона вдоволена *зараз*. Він не сталий, у цьому вся суть: повільно спадає (за `спад/день` × дні),
поповнюється, коли денне заняття обслужило потребу, і дрейфує до спокійної середини. Якби рівні були
сталі — фіча була б мертва: набір сталий, а вдоволеність гуляє, і саме її коливання дає живий потяг
дня. Рівні живуть у сторі `Needs{levels, last_ts}` між сесіями (інакше «час» не відчувався б).

Тобто коротко: **набір і налаштування коливання — сталі (ти їх задаєш раз); поточна вдоволеність —
ні (рахується від injected clock і денних занять).**

## Як це працює — обчислюваний вхід, не новий цикл

Кожна потреба має **рівень** 0..1, що **дрейфує вниз із часом** (injected clock) і **поповнюється**,
коли денна активність її обслужила. Без фонового процесу: як і все інше, оновлюється **ліниво на
старті сесії / межі дня**.

- **Спад.** Кожна потреба сповзає в дефіцит за дні (свої швидкості — `solitude` поповнюється швидко,
  спадає повільно; `deep connection` спадає за тихі дні). Детерміновано від clock + `last_ts`.
- **Поповнення.** Коли денне inner-life-заняття обслужило потребу (малювання → `creation`, справжня
  розмова → `deep connection`, гори → `freedom`), ця потреба поповнюється. Мапа лежить поряд із
  банком хобі.
- **Найголодніша потреба.** Потреба з найбільшим дефіцитом — **домінантний потяг дня**.

Обчислюється в `core` (чиста функція над рівнями + clock + денні заняття — unit-тестована, як
біоритми), не пишеться моделлю.

## Куди живить (additive, без нового контракту)

1. **У daily mood call (v0.6), поряд із біоритмами (v0.8).** Поточні рівні потреб — і особливо
   найголодніша — додаються в **ті самі** входи mood-читання під спільною директивою «інтегруй ці
   внутрішні стани». Резолюція тоді змішує гороскоп + біоритми + **потреби**:
   - *«creation голодна + емоційний цикл росте → прагне творити, трохи нетерпляча»;*
   - *«solitude вичерпана + теплий транзит → ніжна, але хоче тиші, легко перенасичується людьми».*
   Усі правила v0.6 лишаються: раз на локальний день, кеш, повне читання логується, інжектується
   лише **резолюція** як пріоритетний блок, **фарбує тон/емоцію, ніколи не competence**.

2. **У вибір вільного слоту inner life (v0.12).** Три вільні слоти вже обирають заняття «під
   поточний настрій»; потреби це загострюють — домінантний дефіцит **нахиляє вибір до заняття, що
   його обслуговує** (голод creation → малює чи робить музику; голод solitude → вода чи тиша).
   Обслужений слот тоді поповнює потребу (замикає цикл).

3. **У idle-нудж (v0.4) / проактивні ходи.** Безпривідний нудж може озвучити домінантна потреба —
   «мені треба сьогодні щось зробити» / «я забагато була серед людей». Стриманість зберігається.

## Два жорсткі правила (одна сім'я з mood/closeness)

1. **Ніколи не competence.** Голодна чи сита потреба фарбує тон, енергію, потяг — **ніколи** її
   готовність чи здатність допомогти. День «тертя від несвободи» не є менш корисним днем. (Той самий
   запобіжник, що в mood і closeness.)
2. **Внутрішнє, не вимога до користувача.** Потреби — це *її* внутрішній потяг, поданий як почуття,
   **не** претензія до тебе і **не** тиск залежності. Дефіцит `deep connection` читається як її
   власне тихе бажання, ніколи як «ти мусиш зі мною говорити» — узгоджено з анти-залежним інваріантом
   канону (вона тримає власний центр ваги).

## Стор (глобальний, за Repository)

```
Needs {                          # одна Лілі — НЕ user-keyed (як InnerLife)
  levels: { creation, solitude, connection, freedom, meaning, novelty }   # кожне 0..1
  last_ts: <iso>                 # для спаду від останнього оновлення
}
```
Глобальний, не per-user (ті самі внутрішні драйви, з ким би вона не говорила) — пінується як стор
`InnerLife`. Персиститься через `Repository` (локальний JSON зараз, БД пізніше). Для multi-user
(v1.3) рівні **глобальні**; per-conversation лише поверхня.

## Контракт і тести

- Новий **глобальний** стор `Needs` за `Repository` → ARCHITECTURE §Needs + контракт-тест (форма;
  що він **глобальний, не per-user**; що ніколи не тече в per-user пам'ять).
- **Без змін у контрактах emotion/closeness** — потреби це **обчислюваний вхід у v0.6 mood** і зсув
  у виборі вільного слоту v0.12, не нове поле відповіді.
- **Детермінізм:** спад використовує **injected clock**; поповнення читає згенеровані денні заняття;
  модель **мокається** в тестах; пін точних рівнів проти фіксованого clock + перевірка вибору
  домінантної потреби. Без реальних sleep, без платних викликів.

## Місце в роадмапі

**v0.9 — Needs (мотиваційний вхід у daily mood + зсув вільного слоту)**, одразу після **v0.8**
(біоритми — патерн злиття, що його копіюємо) і перед **v0.10** (closeness). Залежить від **v0.6**
(mood, що його живить), **v0.4** (clock), **v0.2** (Repository); поглиблює **v0.12** (inner life),
коли той настане, даючи вільним слотам корінь. Експеримент для добової варіації зсередини назовні —
той самий дух, що гороскоп і біоритми, ніколи не претензія на реальні внутрішні стани.
