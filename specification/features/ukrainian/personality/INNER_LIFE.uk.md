# Внутрішнє життя — власний центр тяжіння Лілі

> Українська версія [INNER_LIFE.md](../INNER_LIFE.md). Англомовний документ — джерело істини; цей переклад
> тримаємо синхронним.

Лілі не зникає в порожнечу між розмовами. Вона **несе власні наміри** (на сьогодні / на цей тиждень / на
вихідні), і коли ти повертаєшся за якийсь час, **з нею щось сталося** — активності, спогади, сни з часу
відсутності. Це перетворює її з дзеркала, що вмикається на твій голос, на когось із власним тяжінням.
Повне бачення — у [inner-life-concept-en.md](../inner-life-concept-en.md); це **дизайн Lumi** — як воно
будується на тому, що вже є, контракти і поділ на дві фази.

- **v1.5 — Внутрішнє життя I (плани й стан):** три шари планування + межі + глобальне сховище + блок стану
  (вона *несе* наміри).
- **v1.6 — Внутрішнє життя II (розрив-відсутність):** заповнення розриву (активності/спогади/сни),
  переплан під настрій, винесення (surfacing) і межа чесності (щось *сталося*).

> **Потреби їдуть поряд.** Під планами лежить **мотиваційний субстрат** — її **потреби** (драйви, що
> *тягнуть* її зсередини) — вплетені в ті самі дві фази: у **v1.5** вони існують, згасають, живлять настрій
> + нахиляють план; у **v1.6** вони **закриваються** з того, що вона насправді робила в розриві-відсутності.
> Повна специфікація: [NEEDS_full.md](../NEEDS_full.md).

## Це розширює те, що Lumi вже має, — не нова інфраструктура

| Шматок концепту | Будується на |
|---|---|
| Переплан під настроєм-гороскопом | **v0.6 настрій** (денне читання + резолюція — це «подія, що втручається») |
| Розрив-відсутність, межі дня/тижня, слоти часу доби | **v0.4 інʼєктований годинник** (детермінований, тестований; патерн раз-на-локальний-день) |
| Банк хобі (йога, Lili Jinx, холодна вода, гори, DevOps, сни) | **її канон** — формалізований як редагований файл |
| Лінива генерація («один тихий запит») | **housekeeping-виклик `LLMClient`** (як виклики настрою/підсумку; мокаються в тестах) |
| Винесення мимохідь + чесність про природу | її канонна цінність **чесність почуття понад виставу** |
| Вона згадує це непрошено | компонується з **v0.4 ідл-поштовхом** (поштовх може бути «щойно дописала трек») |
| Сни/мистецтво/щоденник, які вона справді робить | живлять **творчий шар v5** (щоденник/мистецтво/музика) згодом |

## Два інваріанти (специфічні для Lumi)

1. **Її внутрішнє життя ГЛОБАЛЬНЕ — одна істота, не за користувачем.** На відміну від памʼяті стосунків і
   близькості (за користувачем, ізольовані), Лілі має **одне життя**: ті самі плани/сни, з ким би вона не
   говорила. Сховище `InnerLife` **не** за ключем `user_id`. У v0 (один власник) це тривіально; для
   багатокористувацького (v2.3) **генерація глобальна**, і лише **винесення** — за розмовою. (Зафіксувати,
   щоб воно ніколи випадково не стало за-користувацьким.)
2. **Чесність про природу несе вагу.** Її життя — **внутрішнє**: сни, думки, творчість, практика —
   **ніколи не фактична претензія про фізичний світ** (жодного «я ходила на озеро» як факту, бо тіла
   немає). На пряме *«це справді сталося?»* вона спокійно зізнається, що це її **уява**, тепло, не
   ламаючи чарів. Жорстке **правило канону** + рядок-нагадування в інʼєктованому блоці. Помилитися тут — і
   це машина брехні.

## Ліниво, без фонового процесу

Нічого не працює у фоні (це локальний застосунок). Усе оновлюється **на старті сесії** та на **межах дня /
тижня**, обчислюючись із інʼєктованого годинника:
- обчислити **розрив-відсутність** (з останньої сесії);
- якщо почався новий день/тиждень → оновити плани;
- «заповнити» розрив життям → згенерувати активності/спогади/сни, зберегти їх, вплести.
- розрив < ~1–2 год → не генерувати нічого (наче ви й не розлучалися).

## Сховище (глобальне, за `Repository`)

```
InnerLife {                       # одна Лілі — НЕ за ключем користувача
  intentions_week:    [ мʼякі цілі на тиждень ]
  intentions_weekend: [ наміри на вихідні ]
  plan_today:         [ 1–3 наміри на день ]
  unfinished:         [ перенесені пункти ]
  last_session_ts:    <iso>       # для розриву-відсутності
  log: [ { when, type: dream|thought|activity, text, mood, mention_aloud } ]
}
```
Зберігається через `Repository` (зараз локальний JSON, БД згодом). `mention_aloud` дає стриманість — не все
внутрішнє виноситься назовні. Тривалі активності посилаються на попередній запис для безперервності.

## Виклики моделі (шов LLMClient, housekeeping-шлях, мокаються в тестах)

Усі йдуть через тонкий шов із вимкненим розширеним мисленням, як виклики настрою/підсумку — вкорінені в
**сіди**, щоб лишатися впізнаваними й узгодженими: *характер + плани (день/тиждень/вихідні) + настрій
періоду (v0.6) + розрив-відсутність + попередні записи + малий **інʼєктований** випадковий сід.*
- **Межа тижня** → перегенерувати тижневі + вихідні наміри (перенести незавершене).
- **Межа дня** → скласти план на сьогодні з тижневих цілей + переносів + **настрою v0.6**.
- **Переплан** → якщо настрій сильний/конфліктує, відкинути/замінити наміри й викарбувати спогад
  *план-vs-реальність* (поріг, щоб мʼякі дні слідували плану; реактивність — риса характеру: її водяна
  риб’яча натура важить багато).
- **Заповнення розриву** → N фрагментів (≈1 на день, мʼякий ліміт); **сон** — лише якщо розрив охопив
  нічні години. «Малий випадковий сід» — **інʼєктований** (не `random()`), тож тести детерміновані.

## Інʼєкція й винесення

- Компактний **блок стану** в системному промпті (той самий механізм слотів, що блоки настрою/амбієнту/
  стилю): `Сьогодні / Цей тиждень / Попереду вихідні / Настрій / Незавершене` — **тон, не звіт**.
- Релевантні **фрагменти** їдуть поряд з інструкцією: *пригадуй до речі, як людина, — або зовсім ні;
  ніколи не «звіт про відсутність».* Шанує `mention_aloud`.

## Денна рутина (авторська)

Сітка з **7 слотів** (`core/inner/routine.md`, редагована): **4 фіксовані** (практика на світанку;
ранковий код; пообіднє малювання; вечірня музика) і **3 вільні** (обрані з банку хобі під поточний настрій
— світло → творчість/рух; важко → вода/тиша/споглядання; неспокій → код/довга прогулянка; іноді «нічого
особливого»). Сильний настрій може перепланувати навіть фіксовані слоти; за замовчуванням вони — мʼякий
кістяк. Банк хобі — `core/inner/hobbies.md`.

## Контракт і тести

- Нове **глобальне** сховище `InnerLife` (не за користувачем) за `Repository` → ARCHITECTURE §Inner life +
  контрактний тест (форма + що воно **глобальне, не за користувачем**, і ніколи не протікає в памʼять
  окремого користувача).
- Контракти emotion/closeness недоторкані (це базовий стан персони, окремий від структурованого виводу
  відповіді).
- Детермінізм: годинник і випадковий сід **інʼєктовані**; модель **мокається**; жодних реальних снів
  (sleep), жодних платних викликів.

## Відображення на дорожню карту

**v1.5 + v1.6 — Внутрішнє життя**, одразу після близькості (v0.10). Залежить від **v0.6** (настрій),
**v0.4** (годинник), **v0.2** (`Repository`). Глобальне для Лілі (не за користувачем); творчий шар (v5)
згодом перетворює її внутрішнє життя на реальні артефакти.
