# v0.36 — GitHub Issues

Issues for phase **v0.36 — Lean memory III: facts core + fact recall** (version **v0 — TUI**), derived from
[ROADMAP.md](../../ROADMAP.md) (§v0.36) and
[PROMPT_OPTIMIZATION_II.md](../../../docs/PROMPT_OPTIMIZATION_II.md) (P3 + §4). IDs continue from v0.35
(LUMI-140 → **LUMI-141…143**).

The **facts** tier (~14% of the prompt — the Phase-0 digest + a verbatim tail) is injected wholesale every
turn and is the **only** memory layer with **no pull path** (the v0.31 `recall` searches *messages*). This
phase adds the fact pull path (**embed facts** + `recall(scope=facts)`) and makes facts reach the prompt
**three ways** (mirroring messages): the **always-injected `core`** facts (static), an **auto fact-RAG** block
(per-turn top-K relevant, *push*), and the **`recall(scope=facts)`** tool (on-demand *pull*).

**The `core` flag lifecycle** (the agreed selection mechanism):
- a **`core` boolean** on `LongTermFact`, **persisted in the store** (additive);
- **backfill once** (one model call over all facts → flag ~`LUMI_FACTS_CORE_MAX`) + an **initial guess at
  extraction** (session close tags each new fact);
- **re-flag at session start** over **only the `core=true` pool** → top `LUMI_FACTS_CORE_MAX`, **boundaries
  pinned**, flag written back — this **replaces** the Phase-0 `_ensure_facts_digest` call (**cost-neutral**,
  cheap: input is the small core pool, not all facts);
- **each turn** the prompt injects the `core=true` facts; the tail → `recall(scope=facts)`.

**Hard rules:** **facts are identity** — the `core` facts (esp. **boundaries/agreements**) stay
**always-injected, never tool-gated**; fact hits are **trusted** (her own knowledge — no de-id); **per-user
isolation** (A's facts never surface for B); **off → the Phase-0 digest, byte-identical**; the
`{reply, emotion, intensity}` contract untouched. **No paid calls** in tests (embedder + model mocked).

Depends on **v0.16–17** (`Embedder` + `VectorStore` + auto-RAG), **v0.31** (the `recall` tool it extends),
**v0.34/v0.35** (the lean-memory flag + `core`/index pattern); v1.4 impressions become a later scope.


**Parallelization hints:** **LUMI-141** (pull path) and **LUMI-142** (core flag) are **independent** and can
land in parallel. **LUMI-143** (core static block) and **LUMI-144** (auto-push block) both need 141 + 142 — the
`core` flag (to know what's core), the fact vectors (141), and `recall(scope=facts)`. 143 and 144 are the two
*push* halves (static core + dynamic top-K) and can land in either order; the *pull* is the recall tool (141).
All four are byte-identical until their flags are set (`LUMI_FACTS_CORE_ONLY` / `LUMI_FACTS_RAG`).

## v0.36 — Lean memory III: facts core + fact recall

### LUMI-141 — Fact embedding + `recall(scope=facts)`

**Description:**
The facts layer has no pull path — `recall()` searches *messages*. Add it: embed each `LongTermFact` into the
**same per-user `VectorStore`** as a `kind="fact"` vector, and give `recall` a **`scope`** so she can search
facts by meaning. Reuses the shipped `Embedder` + cosine store + recall tool-loop. (`/core`.)

**What needs to be done:**
- **`VectorRecord` gains `kind`** (`"message"` default | `"fact"`) — additive; update the memory-records
  contract test + ARCHITECTURE §Semantic recall in the same commit.
- **Embed facts** — on write (when a `LongTermFact` is extracted) add a `kind="fact"` `VectorRecord`
  (`Embedder(fact.fact)`, `user_id`-keyed) + a one-off **backfill** over existing facts.
- **`search_vectors` gains a `kind` filter** (none → today's behavior; `"fact"` → only fact vectors).
- **`recall(scope=…)`** — extend the v0.31 tool with `scope = messages | facts | all`; default `messages`
  unchanged; fact hits framed as **trusted recollection** (no de-id), deduped against the prompt. (The
  `thoughts` scope lands in v0.37.)

**Dependencies:** none new (over the shipped Embedder/VectorStore + the v0.31 recall tool).

**Expected result:**
`recall(query, scope=facts)` returns the most relevant long-term facts by meaning, per-user isolated and
trusted. `scope=messages` (default) is byte-identical to today; `scope=all` searches both.

**Acceptance criteria:**
- [ ] `VectorRecord.kind` (additive) defaults to `"message"`; the memory-records contract test updated.
- [ ] Each `LongTermFact` is embedded as a `kind="fact"` vector on write; a backfill embeds existing facts.
- [ ] `recall(scope=facts)` returns only fact vectors; `scope=messages` is unchanged; `scope=all` spans both.
- [ ] **Per-user isolated** — a planted A-fact never returns for B; fact hits are **trusted** (no de-id).
- [ ] **No paid calls** (embedder + model mocked).

### LUMI-142 — The `core` flag + lifecycle

**Description:**
The always-injected identity-core is a **`core` boolean on `LongTermFact`**, persisted in the store. This
issue builds the **flag lifecycle** — backfill, an initial guess at extraction, and a **session-start re-flag**
that re-ranks the core pool to the cap (boundaries pinned), **replacing** the Phase-0 facts-digest call. (No
prompt change yet — that's LUMI-143; here the digest still feeds the prompt while the flags are maintained.)
(`/core`.)

**What needs to be done:**
- **`LongTermFact.core`** (additive boolean, persisted; old records default `False`) — update the
  memory-records contract test in the same commit.
- **Backfill** — a one-off path: one model call over **all** facts → flag ~`LUMI_FACTS_CORE_MAX` as core
  (name, key relationships, **hard boundaries, standing agreements**). Idempotent; per-user.
- **Initial guess at extraction** — the session-close fact extraction (`_accumulate_facts`) tags each *new*
  fact `core` true/false.
- **Session-start re-flag** — over **only the `core=true` pool** (old + new), one model call re-ranks to the
  top `LUMI_FACTS_CORE_MAX` and writes the flag back; **boundaries/agreements pinned** (kept past the cap).
  Wired to **replace** `_ensure_facts_digest` **when `LUMI_FACTS_CORE_ONLY` is on** (LUMI-143); on its own it
  runs alongside, maintaining the flags.
- Config: `LUMI_FACTS_CORE_MAX` (the cap).

**Dependencies:** none new.

**Expected result:**
Every fact carries a persisted `core` flag; the core pool is curated (backfill + extraction) and re-ranked to
`LUMI_FACTS_CORE_MAX` at session start with boundaries always kept. The re-flag's input is the small core pool
(cheap), not all facts.

**Acceptance criteria:**
- [ ] `LongTermFact.core` (additive) persists in the store; old records load as `core=False`.
- [ ] The backfill flags ~`LUMI_FACTS_CORE_MAX` from all facts; extraction sets an initial guess per new fact.
- [ ] The session-start re-flag keeps **≤ `LUMI_FACTS_CORE_MAX`** core facts and **pins boundary/agreement facts** (a unit test plants > N core incl. a boundary → the boundary survives).
- [ ] The re-flag's input is only the `core=true` pool (not all facts); per-user; idempotent.
- [ ] **No paid calls** (model mocked).

### LUMI-143 — Shrink the facts block (core-only) + reconstruction

**Description:**
Wire the `core` flag into the prompt: behind **`LUMI_FACTS_CORE_ONLY`**, the facts block injects the
**`core=true`** facts (≤ `LUMI_FACTS_CORE_MAX`) **instead of** the Phase-0 digest, and the session-start
re-flag (LUMI-142) takes the place of `_ensure_facts_digest`. The episodic tail is reachable via
`recall(scope=facts)` (LUMI-141). Off → the digest, byte-identical. (`/core`.)

**What needs to be done:**
- The prompt builder injects the **`core=true`** facts (capped) when `LUMI_FACTS_CORE_ONLY` is on; **off → the
  Phase-0 digest** (today's block), byte-identical.
- With the flag on, **skip `_ensure_facts_digest`** and run the LUMI-142 **session-start re-flag** instead
  (the cost-neutral swap) — so there's never both a digest *and* a core block, and no broken in-between.
- Config: `LUMI_FACTS_CORE_ONLY` (off) + `LUMI_RECALL_SCOPE` (which scopes the recall tool exposes); document
  in `.env.example`.

**Dependencies:** LUMI-141 (the tail must be pullable), LUMI-142 (the `core` flag must exist).

**Expected result:**
With `LUMI_FACTS_CORE_ONLY` on, the facts block is just the `core` facts (~−10% of the prompt) and the tail is
**retrievable** — a reference to a non-core fact makes `recall(scope=facts)` fire with the right answer. The
identity-core + every boundary stays injected. Off → the full digest, byte-identical.

**Acceptance criteria:**
- [ ] `LUMI_FACTS_CORE_ONLY` on → the facts block is the `core=true` facts (≤ cap, ~−10%); off → the Phase-0 digest, byte-identical.
- [ ] On → `_ensure_facts_digest` is skipped and the re-flag runs instead (no extra model call; no digest+core double-block).
- [ ] **Identity-core + every boundary stays injected** (never tool-gated).
- [ ] **Reconstruction (contract):** core-only + a planted **non-core** fact reference → `recall(scope=facts)` serves it and the answer is correct.
- [ ] Integration: the facts block shrinks **~−10%** core-only; **byte-identical** off. **No paid calls** (embedder + model mocked).

### LUMI-144 — Auto fact-RAG (per-turn push)

**Description:**
The third way facts reach the prompt (beside the always-injected `core` block and the `recall` pull): a
**per-turn push** of the facts most relevant to the incoming message — exactly like the v0.17 message auto-RAG,
but over the fact vectors. A new **`# Релевантні факти`** block in the volatile tail, so relevant tail facts
surface **without her having to call the tool**. (`/core`.)

**What needs to be done:**
- Each turn, embed the incoming message → `search_vectors(scope/kind="fact")` → the **top-`LUMI_FACTS_RAG_K`**
  fact hits above a relevance floor → inject them as a **`# Релевантні факти`** block in the **volatile tail**
  (never the cached prefix).
- **Dedup against the `core` block** — a `core=true` fact already injected is **not** re-pushed; also skip any
  fact whose text is already in the prompt.
- **Trusted** (her own knowledge — no de-id), **per-user isolated** (only her fact vectors); **best-effort** —
  a no-hit / embedder-off degrades to no block, never blocks a turn.
- Config: `LUMI_FACTS_RAG` (off by default) + `LUMI_FACTS_RAG_K` (top-K); document in `.env.example`.

**Dependencies:** LUMI-141 (the fact vectors + the `kind`/scope search), LUMI-142 (the `core` flag, to exclude
core facts from the push).

**Expected result:**
With `LUMI_FACTS_RAG` on, each turn injects the top-K relevant **non-core** facts as a `# Релевантні факти`
block (deduped against the core block); off → no block (byte-identical). So facts surface three ways: core
(static), auto-RAG (push), recall (pull).

**Acceptance criteria:**
- [ ] With `LUMI_FACTS_RAG` on, a `# Релевантні факти` block injects the top-`LUMI_FACTS_RAG_K` relevant fact hits (above the floor) in the **volatile tail**, not the cached prefix.
- [ ] **Core facts are excluded** from the push (deduped against the `core` block — no double-injection); off → no block, byte-identical.
- [ ] **Per-user isolated** (only her fact vectors); **trusted** (no de-id); a no-hit / no-embedder degrades to no block (never blocks a turn).
- [ ] **No paid calls** (embedder + model mocked).

### LUMI-145 — Facts hygiene: `obsolete` flag + `/review-facts` skill

**Description:**
Over time facts accumulate **duplicates, outdated, and irrelevant** entries (you have ~1.2k). Add an
**`obsolete`** flag and an **on-demand review** that marks them — so stale facts stop reaching the prompt while
staying in the store for audit. (`/core`, `/tests`, + a Claude Code skill.)

**What needs to be done:**
- **`LongTermFact.obsolete`** (additive boolean, persisted; default `False`) — update the memory-records
  contract test in the same commit.
- **The filter:** an `obsolete=true` fact is **excluded from every fact path** — the **core** block, the
  **auto fact-RAG** (`# Релевантні факти`), and **`recall(scope=facts)`** — applied at the fact search +
  core-selection points (so all three are covered). It is **kept** in the store (non-destructive, reversible).
- **A `/review-facts` Claude Code skill** (`.claude/skills/`, the `/discover-topics` propose-then-apply
  pattern): reads the per-user facts, finds **duplicates** (cosine-cluster the LUMI-141 fact vectors),
  **outdated** (superseded by a newer fact), and **irrelevant** (ephemeral) — in **batches** (1.2k won't fit
  one call) — proposes the obsolete set **with reasons** for **human review**, then writes `obsolete=true`
  back. **Store-free discipline** (stop the app + back up first); reports counts.
- **Hard guard:** a **`core=true`** fact (esp. boundaries/agreements) is **never auto-obsoleted** — only
  flagged for explicit review.

**Dependencies:** LUMI-141 (the fact vectors for dup-clustering + the search to filter), LUMI-142 (the `core`
flag, to protect core facts from being obsoleted).

**Expected result:**
Running `/review-facts` proposes stale/duplicate facts, you approve, and they're marked `obsolete=true` —
after which they never appear in the core block, the auto fact-RAG, or `recall(scope=facts)`, but remain in the
store. Re-runnable; reversible (un-set the flag).

**Acceptance criteria:**
- [ ] `LongTermFact.obsolete` (additive) persists; old records load as `obsolete=False`; contract test updated.
- [ ] An `obsolete` fact is excluded from **all three** fact paths (core block, auto fact-RAG, `recall(scope=facts)`) — a unit test plants an obsolete fact and asserts it never surfaces; non-obsolete unaffected.
- [ ] The `/review-facts` skill proposes obsolete facts **with reasons** (dup/outdated/irrelevant), applies on approval, is **store-free** (back up first), and **never proposes a `core=true` fact for auto-obsolete** (only flags it for review).
- [ ] Per-user; reversible; **no paid calls in tests** (the skill's review is offline/Opus; the code filter is mock-tested).

## Notes

- **Sequencing.** LUMI-141 (embed + pull) + LUMI-142 (core flag) in parallel → then LUMI-143 (core static block), LUMI-144 (auto-RAG push), and LUMI-145 (obsolete + hygiene skill) in any order. The three prompt paths: 143 = core (static), 144 = auto-RAG (push), 141 = recall (pull); LUMI-145's `obsolete` filter applies to all three.
- **Risk.** This is the **highest-risk** lean-memory tier (facts = identity). The escape hatch is
  `LUMI_FACTS_CORE_ONLY=off` (the Phase-0 digest, byte-identical); boundaries are **pinned** so the cap can
  never drop one.
- **The rest of the lean-memory line.** Thoughts cap + thought-RAG + `recall(scope=thoughts)` = **v0.37**
  (reuses this phase's `kind` discriminator + `scope` seam).
- **Operator.** After enabling `LUMI_FACTS_CORE_ONLY`, run the backfill once (flag the existing ~1.2k facts +
  embed them) — analogous to v0.34's `/regen-summaries`.
