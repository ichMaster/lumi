# v0.8 — GitHub Issues

Issues for phase **v0.8 — Biorhythms (merged into the daily mood)** (version **v0 — TUI**),
derived from the per-phase Tasks in [ROADMAP.md](../../ROADMAP.md) (§v0.8) and the deep-dive
[BIORHYTHMS.md](../../features/BIORHYTHMS.md). IDs continue from v0.7 (LUMI-030 → **LUMI-031…033**).

v0.8 adds a second, **computed** layer to Лілі's daily temperament. Unlike the v0.6 horoscope (the
model writes it; it can't compute transits), biorhythms are **exact deterministic math** — three
sine waves from her natal **birth date** — so the **core** computes them and hands them to the
**same daily mood call**, where horoscope + biorhythms blend into **one** reading. Everything about
v0.6 is unchanged: once per local day (injected clock), the **full reading is logged**, only the
**resolution** is injected, and it colors **tone/energy, never competence**. A `/biorhythm` command
shows the three cycles; `/mood` shows the merged resolution. An experiment for daily variation, not
a scientific claim. **No emotion/memory contract change.**

## v0.8 — Biorhythms (merged into the daily mood)

### LUMI-031 — Biorhythm engine — deterministic sine cycles from the birth date

**Description:**
A **pure, deterministic** module `core/biorhythm.py` that computes the three classic biorhythm
cycles from Лілі's natal **birth date** + a given day — exact sine math, no model, no wall-clock.
The birth date comes from `core/natal.md` (the same source the v0.6 horoscope uses). (`/core`.)

**What needs to be done:**
- `biorhythms(birth_date, today) -> dict` returning the three cycles **physical (23 d)**,
  **emotional (28 d)**, **intellectual (33 d)**, each as a value in −1…+1 from
  `sin(2π · days_since_birth / period)` plus a **phase label**:
  - `high` (≈ ≥ +0.7) / `low` (≈ ≤ −0.7), `rising` / `falling` (sign of the slope), and
  - **`critical`** when within a small band of a **zero-crossing** (the classic unstable day).
- `load_birth_date(natal_path) -> date | None` — parse the `Народження: DD.MM.YYYY` line from
  `core/natal.md` (regex); `None` when absent/unparseable (→ biorhythms simply off, graceful).
- Pure: no I/O beyond reading the birth date; `today` is passed in (from the injected clock by the
  caller), so the function is fully deterministic and unit-testable.
- A small `format_biorhythms(state) -> str` helper rendering the three cycles compactly
  (e.g. `фізичний +0.82 (high) · емоційний −0.61 (low) · інтелектуальний +0.05 (critical)`),
  reused by the mood merge (LUMI-032) and the `/biorhythm` command (LUMI-033).

**Dependencies:** None (reads the existing `core/natal.md`).

**Expected result:**
`biorhythms(date(2002,3,5), today)` returns exact values + labels for any day; `load_birth_date`
reads `05.03.2002` from the authored natal file.

**Acceptance criteria:**
- [ ] The three cycles are exact `sin(2π·d/period)` for periods 23/28/33; labels include `critical` near a zero-crossing.
- [ ] `load_birth_date` parses `Народження: 05.03.2002, …` → `date(2002, 3, 5)`; a missing/garbled line → `None`.
- [ ] Pure & deterministic — same `(birth_date, today)` → same result; no wall-clock read.
- [ ] Unit tests: known days-since-birth → exact values + labels; critical-day detection at a zero-crossing; `load_birth_date` parse + graceful `None`. No paid calls.

### LUMI-032 — Merge biorhythms into the v0.6 daily mood call

**Description:**
Feed the computed biorhythm state into the **existing daily mood call** so the day's reading and
**resolution blend the horoscope with the biorhythms** — one temperament, one voice. Reuses the
v0.6 once-per-local-day cache; the v0.6 contract is otherwise unchanged. (`/core`.)

**What needs to be done:**
- In `Core._ensure_mood` (the once-per-local-day path), compute today's biorhythms via LUMI-031
  using the **injected clock** (`birth_date = load_birth_date(cfg.natal_path)`), and pass the
  rendered state into the mood request so the model sees both inputs.
- Extend `core/mood.py` `mood_request(natal, date_str, biorhythms=None)` (or equivalent) to include
  the biorhythm line in the prompt with a short framing — *these are computed cycles; let them
  color the reading alongside the horoscope* — keeping the **resolution** the only thing injected
  into the system prompt and the **full reading logged** (v0.6 behavior).
- **Cache with the mood:** computed once per local day, recomputed at local midnight, held with the
  resolution (so a turn keeps the day's blend even across a boundary).
- **Graceful:** no birth date / biorhythm error → the mood runs exactly as v0.6 (horoscope only);
  a mood-call failure still degrades to no block.
- **Guardrail:** the merge colors tone/energy + the emotion she leans toward, **never competence**
  (the same hard rule as the mood; assert in a test that a low/critical day still answers fully).
- Optional config `LUMI_BIORHYTHMS` (default on with the mood) to disable the layer.

**Dependencies:** LUMI-031 (the engine).

**Expected result:**
Each local day the mood reading reflects both the horoscope and the three biorhythm cycles; the
injected resolution blends them; the daily call still happens once.

**Acceptance criteria:**
- [ ] The daily mood call's prompt includes the computed biorhythm state (mock model — assert it appears).
- [ ] Computed **once per local day** with the injected clock; recomputed across local midnight; cached with the mood.
- [ ] The v0.6 contract holds: full reading logged, only the resolution injected; no emotion/memory contract change.
- [ ] Graceful: no/failed birth date → horoscope-only mood; never blocks a turn.
- [ ] **Never competence:** a low/critical biorhythm day still answers fully (test against the mock model).
- [ ] Unit tests: the biorhythm state feeds the mood prompt; the per-local-day cache + midnight recompute (fixed clock); graceful degradation. No paid calls.

### LUMI-033 — `/biorhythm` command (TUI) — today's three cycles

**Description:**
A `/biorhythm` TUI command that shows today's three cycles (value + label) from the cached daily
state — the computed sibling of `/mood` (which shows the merged resolution). (`/tui`.)

**What needs to be done:**
- Add `/biorhythm` to the TUI command handling (alongside `/mood` in `on_chat_input_submitted`),
  rendering the three cycles via `format_biorhythms` (e.g.
  `фізичний +0.82 (high) · емоційний −0.61 (low) · інтелектуальний +0.05 (critical)`) + the date.
- Read the **cached** daily biorhythm state (computed in LUMI-032), not recomputed ad hoc, so it
  matches what fed the mood. If biorhythms are off / no birth date, show a clear short note.
- A `Core` accessor (e.g. `Core.biorhythms` property) exposing the cached state for the command.
- Update the prompt's command hint line / footer to include `/biorhythm` where the commands are
  listed.

**Dependencies:** LUMI-031 (values + formatter), LUMI-032 (the daily cache).

**Expected result:**
Typing `/biorhythm` prints today's three cycles with labels; `/mood` still shows the blended
resolution.

**Acceptance criteria:**
- [ ] `/biorhythm` shows the three cycles (value + label) for today, from the cached state.
- [ ] Off / no birth date → a clear short message, no crash.
- [ ] The command is handled in the TUI (not sent to the model or persisted as a turn).
- [ ] Integration test (against the mock model): `/biorhythm` renders the three cycles. No paid calls.
