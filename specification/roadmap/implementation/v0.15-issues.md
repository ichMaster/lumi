# v0.15 — GitHub Issues

Issues for phase **v0.15 — Prompt caching (cache the stable prefix)** (version **v0 — TUI**),
derived from [ROADMAP.md](../../ROADMAP.md) (§v0.15) and
[PROMPT_OPTIMIZATION.md](../../../docs/PROMPT_OPTIMIZATION.md) (§Phase 2). IDs continue from the
drafted Semantic-recall set (LUMI-065 → **LUMI-066…069**).

The whole system prompt is re-billed every turn, but ~85% of it (canon + answer instructions +
the day-stable memory digests + the facts-digest + the mood + the style palette) **barely changes
within a session**. Mark that span as a **cached prefix** (Anthropic prompt caching) so warm turns
pay ~10% for it (5-minute TTL — fine for an active chat). **The same prompt, just ordered for
caching** — no content, character, or contract change.

The cache lives **server-side at Anthropic** (ephemeral, keyed by the exact prefix bytes + the
account); we only **mark** the breakpoint. Caching works only because the facts are now a **digest**
(v0.14… the facts-digest), not 600 growing lines — otherwise the prefix would never be stable.

**The per-turn tail is NOT cached:** the ambient «# Зараз» (its timestamp changes each turn), the
**closeness block** (`update_closeness` recomputes it every turn), the last-24h thoughts, and the
messages. Those move to the end so no per-turn block sits *inside* the cached span.

Depends on **v0.1** (the `LLMClient` seam) and **v0.3** (the assembled prompt + `ResponseStats`) —
both shipped. **No contract change** (the emotion field + memory records are untouched; `cache_prefix`
is an additive, optional hint on the seam, ignored by non-caching backends).

## v0.15 — Prompt caching

### LUMI-066 — Reorder `build_system_prompt`: stable prefix vs per-turn tail

**Description:**
Reorder the system prompt so all **stable-within-a-session** blocks form a contiguous prefix and the
**per-turn** blocks trail — then expose where the prefix ends, so the adapter can mark it. Same
content, only order changes. (`/core`.)

**What needs to be done:**
- **Reorder** the assembly in `build_system_prompt` (`core/prompt.py`):
  - **Stable prefix** (in order): canon + reasoning directive + `# Як відповідати` (emotion/relation
    instructions — static constants) + `# Памʼять про цю людину` (weeks/days/sessions/**facts-digest**/digest) +
    `# Настрій дня` (mood, locked per day).
  - **Per-turn tail** (after the prefix, before the messages): `# Зараз` (ambient now/here — moves
    from its current **early** position at ~`prompt.py:189`) + the **closeness block** (recomputed
    each turn) + `# Що в мене на думці` (last-24h thoughts) + `# Стиль відповіді` (the style palette —
    **kept last** for salience; it's static but rides in the tail so it stays the most-salient block).
- **Expose the boundary:** return the full `system` string **and** the cacheable prefix — e.g.
  `build_system_prompt(...) -> (system, cache_prefix)` where `cache_prefix` is exactly the stable
  prefix substring (so `system.startswith(cache_prefix)` holds). Keep the **full text byte-identical
  in content** to today (only block order changes); update `core/agent.py::_system_prompt` + any
  callers + `last_prompt`.
- The blocks are **order-independent** (same authored content), so **no behavior change**; `/prompts`
  shows the reordered prompt.

**Dependencies:** None (builds on the existing assembly).

**Expected result:**
`build_system_prompt(...)` returns `(system, cache_prefix)`; the per-turn blocks (ambient, closeness,
thoughts) are all **after** `cache_prefix`; `system == cache_prefix + tail` (nothing lost or duplicated).

**Acceptance criteria:**
- [ ] Ambient/closeness/thoughts/style are assembled **last** (the tail, style last); canon/instructions/memory/facts-digest/mood form the prefix.
- [ ] `build_system_prompt` exposes the `cache_prefix` (a true prefix of `system`); no content is dropped or duplicated (reassembly is byte-identical).
- [ ] No per-turn block appears inside `cache_prefix`.
- [ ] Unit: the prefix/tail split; prefix contains none of {ambient, closeness, thoughts}; full-text content unchanged vs the old order (same blocks).

### LUMI-067 — Cache breakpoint in the Anthropic adapter + `LUMI_PROMPT_CACHE`

**Description:**
Teach the `LLMClient` seam to mark a cache breakpoint, and have the Anthropic adapter emit the
`system` as content blocks with `cache_control` on the prefix. Additive and provider-agnostic —
other backends ignore it. (`/core`.)

**What needs to be done:**
- **Seam** (`core/llm.py`): add an optional `cache_prefix: str | None = None` to `reply` /
  `reply_structured` (Protocol + impls). The core passes the full `system` + the `cache_prefix` from
  LUMI-066 on the **main reply turn only** (housekeeping calls — mood/summaries/facts-digest — pass
  none).
- **AnthropicClient** (`_base_kwargs`): when `cfg.prompt_cache` is on **and** `cache_prefix` is set,
  pass `system` as **content blocks** —
  `[{"type":"text","text":<prefix>,"cache_control":{"type":"ephemeral"}}, {"type":"text","text":<remainder>}]`
  (remainder = `system[len(cache_prefix):]`). Otherwise pass `system` as a plain string (today's
  behavior, byte-identical).
- **Config** (`core/config.py`): `prompt_cache` (`LUMI_PROMPT_CACHE`, **on** by default).
- **Other backends:** `MockLLMClient` (and any future OpenAI/DeepSeek adapter) **ignore** `cache_prefix`
  — they use `system` as a plain string, so the assembled text is identical with or without caching.

**Dependencies:** LUMI-066 (the `cache_prefix`).

**Expected result:**
With `LUMI_PROMPT_CACHE=on`, the Anthropic request's `system` is a 2-block list with `cache_control`
on the prefix block; off → a plain string; the mock ignores it entirely.

**Acceptance criteria:**
- [ ] `reply` / `reply_structured` accept an optional `cache_prefix`; passed only on the main reply turn.
- [ ] On + prefix → `system` is `[prefix-block(cache_control:ephemeral), remainder-block]`; the two blocks concatenate to the original text.
- [ ] Off / no prefix → `system` is the plain string (unchanged).
- [ ] `LUMI_PROMPT_CACHE` config (on by default); `MockLLMClient` ignores `cache_prefix`.
- [ ] Unit tests (fake Anthropic client, no network): block shape on/off; mock ignores the hint. No paid calls.

### LUMI-069 — Caching **contract test** (prefix stability + per-turn isolation)

**Description:**
Pin the invariants that make caching correct, so a future change can't silently break the cache or
leak a per-turn block into the prefix. (`/tests`.)

**What needs to be done:**
- `tests/contract/test_prompt_cache.py`:
  - **Prefix stability:** across consecutive turns of one session (with the **ambient time, closeness
    level, and thoughts changing**), the `cache_prefix` is **byte-identical** turn-to-turn.
  - **Per-turn isolation:** a per-turn block (ambient timestamp / closeness / a new thought) **never**
    appears inside `cache_prefix` (only in the tail).
  - **Marker on/off:** `LUMI_PROMPT_CACHE=on` → the Anthropic request carries the `cache_control`
    breakpoint; off → no marker, plain-string `system`.
  - **Mock-agnostic:** the `MockLLMClient` ignores the hint and returns an **identical** reply
    (the emotion-field contract test still passes verbatim).
- Use a fixed clock + fake ambient/closeness so the only changing inputs are the tail; **no network,
  no paid calls**.

**Dependencies:** LUMI-066, LUMI-067.

**Expected result:**
A single contract module proves the cache prefix is stable + isolated and the marker toggles — and
fails loudly if a future change lets a per-turn block drift into the cached span.

**Acceptance criteria:**
- [ ] Contract: `cache_prefix` byte-identical across turns while ambient/closeness/thoughts change.
- [ ] Contract: no per-turn block ever inside `cache_prefix`.
- [ ] Contract: marker present when on / absent when off; mock ignores it (identical reply).
- [ ] All via the mock backend (no network, no paid calls); green in CI.

## Definition of Done (phase)

Mirrors [ROADMAP.md §v0.15](../../ROADMAP.md): within a session the system-prompt **prefix** (canon +
instructions + day-stable memory + facts-digest + mood + style) is **byte-identical turn-to-turn** and
carries one `cache_control` breakpoint; the **per-turn blocks** (ambient time, the closeness block —
recomputed each turn, thoughts) and the messages sit **after** it; with caching on, a warm turn reports
`cache_read` tokens covering the prefix; off → the same prompt text, no marker. **Provider-agnostic**
(the mock + future backends ignore the hint), **mocked in every test** (no network / paid calls), and
**no content, character, or contract change** (the emotion field + memory records are untouched). It
relies on the **facts digest** to keep the prefix stable.
