## Notes

- **LUMI-180** = the foundation: `_set_busy` stops disabling the widget (when `LUMI_INPUT_BUFFER` on);
  the command/directive/chat dispatch is extracted into a reusable `_handle_input_line` for both the
  live path and the drain. Behavior-preserving refactor; off → byte-identical.
- **LUMI-181** = the queue: a submit-while-busy appends to a `deque` and echoes the user row now (no
  turn), with an `echoed` marker so the eventual turn won't double-render.
- **LUMI-182** = the payoff: on turn completion the next queued line runs — **one turn per message,
  FIFO, never merged** (distinct from the v0.13 Telegram burst-consolidation).
- **LUMI-183** = the finish: proactive workers (nudge/think/scheduler/inbox) gain a queue-empty guard
  so a proactive turn never jumps the queue; a `⋯{n}` pending indicator; `LUMI_INPUT_BUFFER` docs; the
  byte-identical-off pin.
- **No core change, no contract change** — pure TUI/interface phase; the model stays single-turn
  serialized (`_busy` mutex); **off (`LUMI_INPUT_BUFFER`) → byte-identical** (pinned per issue); no
  paid calls in tests.
