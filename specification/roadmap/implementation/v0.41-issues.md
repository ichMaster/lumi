### LUMI-160 â€” Profile config: `DEFAULT_MODEL_PROFILES` + the `LUMI_MODEL_PROFILES` parser

### LUMI-161 â€” `Core.switch_profile(name)`: atomic engine + tier swap

**Description:**
The core seam. `switch_profile` resolves a named profile and, **in one step**: rebuilds the client via
the v0.37 factory (`switch_model` â€” provider + reply model) **and** re-points the three v0.40 tier
fields (`_model_think` / `_model_mood` / `_model_housekeeping`) to the profile's ids. **Atomic on
failure**: if the factory raises (unknown provider / missing key), the old client **and** the old tiers
stay untouched. The active profile name is exposed for the status bar. With a matching profile the
v0.40 provider guard stops blocking (the tier ids always belong to the active provider), so routing
now works on every engine.

**What needs to be done:**
- `Core.switch_profile(name)`: resolve from the configured profiles (unknown â†’ a clear `ValueError`,
  the `resolve_model_target` style); build the new client **first**, then assign client + `_model` +
  `_active_provider` + the three tier fields together; set `self._active_profile = name`.
- A `profile` property (the active name, or `None` when running on raw env vars); `switch_model`
  (the reply-only path) **clears** the active profile name (the stack no longer matches a named set â€”
  the tiers keep their current values).
- Wire `model_profiles` from config through `build_core`.

**Dependencies:** **LUMI-160** (the profile config); reuses v0.37 (`switch_model` + factory) and
v0.40 (`_model_for` â€” untouched, it just reads the re-pointed fields).

**Expected result:**
`switch_profile("gemini")` moves the engine to Gemini **and** routes think/mood/housekeeping to the
flash tiers (verified per-kind by the mock); a raising factory leaves engine + tiers exactly as they
were; `switch_profile("anthropic")` restores the Claude set.

**Acceptance criteria:**
- [ ] `switch_profile` re-points the client + reply model + all three tiers in one call (mock factory + `MockLLMClient` record per-kind models).
- [ ] **Routing works on a foreign engine under its profile:** on the gemini profile, a housekeeping call carries the profile's flash-lite id (the guard doesn't block a matching provider).
- [ ] **Atomic failure:** a raising factory leaves client, `_model`, provider, and all tiers unchanged.
- [ ] Unknown profile â†’ a clear, non-fatal error; profiles unset â†’ the v0.40 behavior byte-identical (existing routing tests pass verbatim).
- [ ] **No paid calls** (mock factory/client).

### LUMI-163 â€” `/model <full-id>`: bare-id provider inference

- Document the bare-id form in the `/model` help text + MODELS_SETUP.md (one line each).

**Dependencies:** none â€” extends the v0.37 resolver; independent of profiles (runs in parallel).

**Expected result:**
`/model claude-haiku-4-5-20251001` switches to Anthropic Haiku without an alias; `/model gpt-5.5-mini`

**Acceptance criteria:**
