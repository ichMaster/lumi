## v0.11 вЂ” Face variants & mood themes

### LUMI-042 вЂ” Variant resolver (random, no-repeat) + extended signal parse

**Description:**
Extend the v0.7 viewer resolver so each emotion is a **folder** of images and the viewer shows a
**random** one (no immediate repeat). The face signal parse gains the **theme**:
`<theme> <emotion> <intensity>` вЂ” a bare `<emotion> <intensity>` still works (the default theme).
(`/viewer`.)

**What needs to be done:**
- Resolve `(theme, emotion)` в†’ gather `faces/<theme>/<emotion>/*.png` and pick one at **random,
  with no immediate repeat**; `FaceSwitcher` remembers the last pick and re-picks on emotion change
  (an optional coarse interval for liveliness is fine).
- **Total over the enum, `calm`/default fallback:** a missing emotion в†’ the theme's `calm/`; a
  missing theme в†’ the default theme; an empty set в†’ the v0.7 single `faces/<emotion>.png`. The
  window never breaks.
- Extend the signal parse to accept `<theme> <emotion> <intensity>`; a bare `<emotion> <intensity>`
  (no theme) в†’ the default theme. `intensity` is parsed as before (it need not vary the picture).

**Dependencies:** None (builds on the v0.7 viewer + signal).

**Expected result:**
For the same emotion the viewer shows a **different picture among several** (no immediate repeat),
selected from `faces/<theme>/<emotion>/`; the signal carries the theme.

**Acceptance criteria:**
- [ ] Random variant pick over `faces/<theme>/<emotion>/*.png`, **no immediate repeat**; re-pick on emotion change.
- [ ] Fallback chain: missing emotion в†’ theme `calm/`; missing theme в†’ default theme; empty в†’ v0.7 single image. Never crashes.
- [ ] Signal parse accepts `<theme> <emotion> <intensity>` **and** the bare `<emotion> <intensity>` (в†’ default theme).
- [ ] Unit tests against a **fake faces tree** (random + no-repeat, total over the enum, fallbacks, signal parse). No real images / no paid calls.

### LUMI-043 вЂ” Theme manifest (`faces/themes.md`) + folder auto-discovery

**Description:**
An editable theme manifest `faces/themes.md` вЂ” each theme name + a **one-line description** (so the
mood can choose) + the **default theme**; theme folders are **auto-discovered** under `faces/`.
(`/viewer`.)

**What needs to be done:**
- A loader for `faces/themes.md` в†’ `{theme: description}` + the **default theme** name.
- **Auto-discover** theme folders (a subfolder of `faces/` that contains emotion folders). The
  manifest names/describes them; a folder without a manifest entry still works (no description).
- A **missing manifest** degrades to the default/flat v0.7 behavior (no themes).

**Dependencies:** None.

**Expected result:**
`load_themes(faces_dir)` yields the theme descriptions + the default theme; theme folders are found
automatically.

**Acceptance criteria:**
- [ ] Manifest loader returns `{theme: one-line description}` + the **default theme**.
- [ ] Theme folders auto-discovered under `faces/`; a missing manifest в†’ default/flat behavior.
- [ ] Unit tests against a fake faces tree + a sample `themes.md` (parse, default, discovery, missing file). No paid calls.

### LUMI-044 вЂ” Mood picks the theme + core writes the themed face signal

**Description:**
The **daily mood call (v0.6)** also returns a **theme** from the manifest that fits the day;
`MoodState` gains a `theme`; cached per local day, recomputed at local midnight; graceful (no/failed
mood в†’ the **default theme**). The core writes `<theme> <emotion> <intensity>` to the face signal.
(`/core`.)

**What needs to be done:**
- The mood call returns a `theme` chosen from the manifest descriptions (LUMI-043); validate it
  against the known themes (unknown в†’ default theme).
- `MoodState` gains `theme`; cached **once per local day** with the mood, recomputed at local
  midnight (the injected clock); a no/failed mood (or no themes) в†’ the **default theme**.
- `Core._write_face_signal` writes `<theme> <emotion> <intensity>` (the v0.7 signal extended with
  the theme вЂ” the format LUMI-042 parses).
- Never blocks a turn (theme selection degrades to default, like the mood).

**Dependencies:** LUMI-042 (the signal format), LUMI-043 (the theme manifest).

**Expected result:**
The day's outfit/theme is chosen by the mood (stable within a day, re-picked at midnight) and ridden
in the face signal; the viewer renders `faces/<theme>/<emotion>/вЂ¦`.

**Acceptance criteria:**
- [ ] The daily mood call returns a manifest **theme**; `MoodState.theme` cached per local day, recomputed at midnight; unknown/failed в†’ default theme.
- [ ] `_write_face_signal` emits `<theme> <emotion> <intensity>`; a no-theme state still emits the bare signal.
- [ ] Theme selection **never blocks the turn** (degrades to default); the v0.6 mood resolution + the locked emotion contract are unchanged.
- [ ] Unit tests: moodв†’theme selection (mock model + fixed clock; cached per day, recompute across midnight; default on failure); the extended signal write. No paid calls.
