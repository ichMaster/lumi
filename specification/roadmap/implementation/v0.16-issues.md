# v0.16 — GitHub Issues

Issues for phase **v0.16 — Semantic recall I: index & search (RAG foundation)** (version **v0 —
TUI**), derived from [ROADMAP.md](../../ROADMAP.md) (§v0.16) and
[SEMANTIC_RECALL.md](../../features/SEMANTIC_RECALL.md). IDs continue from v0.14 (LUMI-060 →
**LUMI-061…065**).

Lumi's memory so far is **lossy by design** (window → summaries/gists → facts + impressions) — none
can pull **the exact words you said months ago** when they suddenly matter. Semantic recall adds the
missing layer: **every message is embedded into a per-user vector store and retrieved by meaning**.
v0.16 lays the foundation — two seams + the index + an explicit **`/recall <query>`** search; the
automatic per-turn RAG rides on it in **v0.17**.

Two seams the core depends on (never an SDK), both **mockable** (no paid APIs in CI):
- **`Embedder`** (mirrors `core/llm.py`'s `LLMClient`): `embed(texts) → vectors`. **Default local
  multilingual** (Ukrainian-capable; private — text never leaves the machine), **cloud optional** via
  config (Voyage/OpenAI), `MockEmbedder` with deterministic fake vectors for tests.
- **`VectorStore`** behind the `Repository`, **keyed by `user_id`**: `{user_id, msg_id, vector, text,
  ts, role}`; local brute-force numpy cosine (instant at this scale). Same **isolation invariant** as
  the rest of memory — retrieval runs only over the requesting user's vectors (contract test).

Depends on **v0.2** (messages + the Repository) — both shipped; **no dependency** on the inner-life /
emotional-memory phases (v0.19–v0.23). No core **contract** change (the emotion field
`{reply, emotion, intensity}` is untouched); the Repository Protocol gains additive vector methods.

## v0.16 — Semantic recall I

### LUMI-061 — `Embedder` seam (`core/embedder.py`) + `MockEmbedder`

**Description:**
The embedding seam — the only way the core turns text into vectors, mirroring the `LLMClient` seam
(`core/llm.py`): the core depends on the `Embedder` Protocol, **never** an embedding SDK. Default is a
**local multilingual** model (private, no per-call cost); a cloud API is opt-in. (`/core`.)

**What needs to be done:**
- `core/embedder.py`: an `Embedder` **Protocol** — `embed(texts: list[str]) -> list[list[float]]`
  (batch; returns one unit-normalized vector per text) and a `dim` property.
- `LocalEmbedder` — a **local multilingual** model (e.g. `fastembed` / `sentence-transformers`,
  Ukrainian-capable), **lazy-imported** so the module + `MockEmbedder` import **without** the model
  installed. Messages **never leave the machine**.
- `CloudEmbedder` (optional) — Voyage / OpenAI via config; sends text out, so it's **off unless
  configured**; the key is a **secret** (`VOYAGE_API_KEY` / `OPENAI_API_KEY`, never logged/committed).
- `MockEmbedder` — **deterministic** fake vectors (e.g. seeded from a hash of the text) so cosine
  ranking is testable; records calls; **no network, no paid APIs**.
- **Config** (`core/config.py`): `recall` (`LUMI_RECALL`, **off** by default — the whole feature),
  `embed_provider` (`LUMI_EMBED_PROVIDER` = `local` | `voyage` | `openai`, default `local`),
  `embed_model` (`LUMI_EMBED_MODEL`, a sensible multilingual default), the cloud key (secret).
- `[project.optional-dependencies]` **`embed`** group in `pyproject.toml` (like `telegram` / `voice`)
  — `fastembed` (or `sentence-transformers`) + `numpy`; the core/tests import without it (lazy).

**Dependencies:** None (builds on v0.2 messages).

**Expected result:**
`import core.embedder` works **without** the embedding model installed; `MockEmbedder().embed(["hi",
"hello"])` returns two deterministic, same-`dim` vectors; identical text → identical vector.

**Acceptance criteria:**
- [ ] `Embedder` Protocol (`embed(texts) -> vectors`, `dim`); `import core.embedder` works without the model (lazy).
- [ ] `LocalEmbedder` is the default; messages never leave the machine; `CloudEmbedder` is opt-in and its key is a secret (never logged).
- [ ] `MockEmbedder` returns **deterministic** vectors (same text → same vector) and records calls.
- [ ] Config: `LUMI_RECALL` (off), `LUMI_EMBED_PROVIDER` (local default), `LUMI_EMBED_MODEL`, cloud key (secret); `embed` optional dep group.
- [ ] Unit tests via `MockEmbedder` (no network, no paid calls).

### LUMI-062 — `VectorStore` seam (`Repository` + `JsonRepository`) — per-user, numpy cosine

**Description:**
The per-user vector store, behind the `Repository` (the same principle as the rest of memory —
swapping the backend never touches the core). Local brute-force cosine is instant at this scale; a
server vector DB lands later. (`/state`, `/core` contract.)

**What needs to be done:**
- **Data shape** (`core/repository.py`): a `VectorRecord` dataclass `{user_id, msg_id, vector, text,
  ts, role}` (matches SEMANTIC_RECALL.md §seams). `msg_id` is a **stable, content-addressed** id
  derived from the message (e.g. a hash of `session_id|ts|role|text`) so backfill/index is idempotent
  (re-embedding the same message never duplicates).
- **Repository Protocol** (additive — no existing method changes): `add_vector(rec: VectorRecord)`,
  `has_vector(user_id, msg_id) -> bool` (for incremental/backfill), `search_vectors(user_id,
  query_vector, k) -> list[tuple[float, VectorRecord]]` (top-K cosine, **scoped to `user_id`**), and
  **extend `clear_memory(user_id)`** to drop that user's vectors too (privacy — `/forget` clears them).
- **`JsonRepository`** (`state/local_store.py`): implement the four — vectors persisted per-user;
  `search_vectors` is brute-force **numpy cosine** over only that user's vectors, returning the top-K
  by score (descending). Empty/cold store → `[]`.
- **Isolation built in:** every read/write takes `user_id`; `search_vectors` filters to the owner
  before ranking — A's vectors are never in B's candidate set (pinned by LUMI-065).

**Dependencies:** None (the store is independent of the embedder — it holds and ranks vectors).

**Expected result:**
`add_vector` then `search_vectors(user_id, q, k)` returns that user's closest records by cosine,
descending; `has_vector` is `True` only after the record is added; `clear_memory` removes them.

**Acceptance criteria:**
- [ ] `VectorRecord{user_id, msg_id, vector, text, ts, role}` with a stable content-addressed `msg_id`.
- [ ] `Repository` gains `add_vector` / `has_vector` / `search_vectors` (additive); `JsonRepository` implements them (numpy cosine, per-user).
- [ ] `search_vectors(user_id, …)` ranks **only** that user's vectors, top-K descending; cold store → `[]`.
- [ ] `clear_memory(user_id)` also clears that user's vectors; other users untouched.
- [ ] Unit tests: round-trip, top-K ranking on deterministic vectors, `has_vector`, clear. No paid calls.

### LUMI-063 — Index on write + one-time backfill (`core/agent.py`)

**Description:**
Every message (yours **and** Лілі's) is embedded as it's written; existing messages are **backfilled**
once; incremental thereafter. Best-effort — an embed failure never blocks or loses a turn (the message
is still stored, just not yet indexed). (`/core`.)

**What needs to be done:**
- **Index on write:** after the two `append_message` calls in `Core.reply` (`core/agent.py` ~L986–999),
  embed each new message and `add_vector` it (guarded by `LUMI_RECALL`). Wrap in try/except —
  on an embedder error, **log and continue** (the message is stored; indexing is retried by backfill).
- **Backfill:** a `backfill_vectors()` pass (lazy, at startup / first recall) that walks the user's
  stored messages and embeds any whose `msg_id` is **not** in the store (`has_vector` → skip) — so a
  cold store catches up once, and it's incremental forever after. Batched embed; bounded work per pass.
- **Degrade gracefully:** with `LUMI_RECALL=off` or no embedder, indexing is a no-op; the turn is
  unchanged (no new latency on the critical path — embedding is best-effort/after the reply is built).

**Dependencies:** LUMI-061 (the `Embedder`), LUMI-062 (the `VectorStore`).

**Expected result:**
After a few turns, both your and Лілі's messages are in the per-user vector store; restarting on an
un-indexed history backfills it once; an embedder error leaves the chat working and the message stored.

**Acceptance criteria:**
- [ ] Each new message (user + lili) is embedded + stored on write when `LUMI_RECALL=on`.
- [ ] **Backfill** indexes only messages without a vector (`has_vector`); idempotent (re-run adds nothing).
- [ ] An embedder failure is caught — the turn still completes and the message is still persisted (retried later).
- [ ] `LUMI_RECALL=off` / no embedder → indexing is a no-op; turn behavior unchanged.
- [ ] Unit tests via `MockEmbedder` + an in-memory store: index-on-write, backfill idempotency, error degradation. No paid calls.

### LUMI-064 — `/recall <query>` — explicit top-K semantic search

**Description:**
The "search on request" surface: embed the query → top-K cosine over **this user's** vectors → the
closest past lines, **dated**. Useful on its own, and the proof the index works before automatic RAG
(v0.17) rides on it. (`/tui`, `/core`.)

**What needs to be done:**
- **Core** (`core/agent.py`): `Core.recall(query: str, k: int | None = None) -> list[tuple[float,
  VectorRecord]]` — embed the query via the `Embedder`, `search_vectors(self._user_id, q, k)`
  (defaulting `k` to `LUMI_RECALL_K`), return the dated matches. Runs a backfill first so a cold
  store still answers. Empty store / no embedder → `[]` (never raises).
- **TUI** (`tui/app.py`): a `/recall <query>` command (next to `/mood` ~L459 and in the prompt
  subtitle hint ~L246) that calls `Core.recall` and renders each hit as a **dated, scored** line
  (e.g. `2026-03-12 · «…» (0.81)`); an empty query or no results prints a friendly note.
- **Config:** `recall_k` (`LUMI_RECALL_K`, default e.g. 5).

**Dependencies:** LUMI-061, LUMI-062 (uses LUMI-063's index when present; unit-tested standalone).

**Expected result:**
`/recall кава` returns the past lines most about coffee for **this** user, dated and ranked — even if
they're long out of the verbatim window.

**Acceptance criteria:**
- [ ] `Core.recall(query, k)` embeds the query and returns this user's top-K by cosine (dated), never raising.
- [ ] `/recall <query>` renders dated, scored hits; empty query / no matches → a friendly note.
- [ ] Scoped to the current user (no cross-user results — pinned by LUMI-065).
- [ ] `LUMI_RECALL_K` config; off/no-embedder → graceful empty.
- [ ] Unit tests via `MockEmbedder` + store: ranking order, dated render, empty/error paths. No paid calls.

### LUMI-065 — Per-user isolation **contract test** + graceful degradation

**Description:**
The hard rule, pinned: retrieval (both `/recall` and the per-turn RAG to come) runs **only** over the
requesting user's vectors — A's messages can **never** surface for B, the same isolation invariant as
the rest of memory. Plus the graceful-degradation guarantees as contract-level checks. (`/tests`.)

**What needs to be done:**
- `tests/contract/test_semantic_recall_isolation.py` (beside the existing memory-isolation contract
  tests): seed user A and user B with distinct messages/vectors; assert `search_vectors(B, …)` and
  `Core.recall` for B **never** return any of A's records, and vice-versa — across `add_vector`,
  backfill, and `/recall`.
- **Degradation contract:** assert an `Embedder` that raises leaves `reply` working and the message
  stored (LUMI-063), and that `recall` returns `[]` rather than raising (LUMI-064).
- **`clear_memory` contract:** after `clear_memory(A)`, A's vectors are gone and B's are intact.
- Reuse the project's `MockEmbedder` + a fresh per-user store; **no network, no paid calls**.

**Dependencies:** LUMI-062, LUMI-063, LUMI-064.

**Expected result:**
A single contract module proves the isolation invariant for the vector layer and the graceful-failure
guarantees — and fails loudly if a future change lets vectors cross users.

**Acceptance criteria:**
- [ ] Contract: `search_vectors` / `Core.recall` return **only** the requesting user's vectors (A↔B, both directions).
- [ ] Contract: an embedder error never breaks a turn or `recall` (message still stored; `recall` → `[]`).
- [ ] Contract: `clear_memory(A)` removes A's vectors only; B's remain.
- [ ] All via `MockEmbedder` (no network, no paid calls); green in CI.
