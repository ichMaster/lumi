# v0.33 — GitHub Issues

Issues for phase **v0.33 — Thought-tools: the autonomous mind *acts*** (version **v0 — TUI**), derived
from [ROADMAP.md](../../ROADMAP.md) (§v0.33), [TOOL_THOUGHTS.md](../../features/TOOL_THOUGHTS.md),
[FILE_THOUGHTS.md](../../features/FILE_THOUGHTS.md), and [THOUGHT_STREAM.md](../../features/THOUGHT_STREAM.md).
IDs continue from v0.32 (LUMI-125 → **LUMI-126…135**).

Лілі's `%directives` (the v0.12 thought-stream — `core/thoughts.py` `Directive`/`REGISTRY`, `Core.think` /
`run_directive` / `resolve`) gain the ability to **run tools in the *think* path**, so her autonomous mind
can *act, find out, make, keep up, and do what you ask* — not only reflect. **One new seam** (the bounded
tool-loop with the **thought terminal** — free text + `ЕМОЦІЯ`, never `set_state`), built once and reused by
every directive family.

**The one new mechanism.** Today a thought is a **single tool-less** call (`Core.think` →
`_housekeeping_reply` ending in `ЕМОЦІЯ:`); the file/wiki/image/news/web tools live in the **reply** loop
whose terminal is `set_state`. This phase runs the **bounded tool-loop with tools available but the *thought*
terminal**, plus two infra cleanups (a **table-driven `Directive` record** + an **extended placeholder
resolver**), the **de-identified thought-driven query** rule (stricter than the reply-path rule — a new
contract test), and all the directives as **thin flag-gated rows**.

**Hard rules (unchanged):** the thought terminal **holds** — never `set_state` inside a thought; tool
**results stay untrusted** (except `%recall` — *her own* memory, trusted); the thought-driven external query
carries **no personal data** (de-identified; `%prompt` exempt as the owner authored it); **off by default,
per family** → that family is **absent** and the v0.12 tool-less think is unchanged; the `{reply, emotion,
intensity}` contract + per-user isolation are untouched. **Every tool is mocked** in tests (no
network/key/paid/Telegram).

The tools themselves are all shipped: file (v0.19/0.20/**0.32**), wiki (v0.21), image (v0.22–24), news
(v0.25), journal (v0.28), recall (v0.31), web (v0.27). Depends on **v0.12** (the engine + `run_directive` +
placeholders), **v0.3** (the validated emotion the thought parse reuses).

## v0.33 — Thought-tools: the autonomous mind acts

### LUMI-126 — Think-path tool-loop + thought terminal + table-driven `Directive`

**Description:**
The one new mechanism: run the **bounded tool-loop in the *think* path**, with tools available but the
**thought terminal** (free text + `ЕМОЦІЯ:`, never `set_state`). Built once in `core`, reused by every
family. Plus the **table-driven `Directive` record** that lets each family be a thin row. (`/core`.)

**What needs to be done:**
- A **think-path tool-loop** (`core/`) — like the reply loop's `_tool_loop` (core/llm.py) but terminating on
  a **final free-text thought ending in `ЕМОЦІЯ:`** (parsed by `parse_thought`), **never** `set_state`. It
  offers the directive's allowed tools, applies the reply loop's **per-turn caps + the v0.19 trace**, and
  records a `Thought` as today's `Core.think` does. A directive with `tools=()` runs the **existing tool-less
  path unchanged**.
- **Optimize the `Directive` record** (`core/thoughts.py`) — add `tools` (the tool families it may use),
  `cap` (per-directive tool-call cap), `surface` (surfacing default), `trigger` (scheduler default), and
  `instruction_from_topic` (for `%prompt`: the topic *is* the instruction). The loop is **table-driven** off
  these fields. `%think` / `%wonder` keep `tools=()` and behave exactly as in v0.12.
- **Config:** `LUMI_THOUGHT_TOOLS` (master gate for *tool-using* thoughts — off by default); with it off, no
  directive offers tools and `think` is the v0.12 tool-less call.

**Dependencies:** none new (over the shipped v0.12 engine + v0.19 tool-loop).

**Expected result:**
With `LUMI_THOUGHT_TOOLS=on` (and a family flag), a directive can call tools mid-thought and still terminate
in a thought (free text + `ЕМОЦІЯ`), recording a `Thought`; `set_state` is never reachable inside a thought.
Off → `think` is the v0.12 tool-less call, byte-identical.

**Acceptance criteria:**
- [ ] A think-path tool-loop offers a directive's tools and **terminates on `ЕМОЦІЯ:` free text, never `set_state`**; records a `Thought`; carries the per-turn caps + trace.
- [ ] `Directive` gains `tools` / `cap` / `surface` / `trigger` / `instruction_from_topic` (table-driven); `%think` / `%wonder` keep `tools=()`, unchanged.
- [ ] `LUMI_THOUGHT_TOOLS` (off) gates the tool path; off → the v0.12 tool-less `think` is byte-identical.
- [ ] Unit tests: a stub-tool directive loops then terminates in a thought; **no `set_state` inside a thought**; `tools=()` → the old path; the emotion parse holds. **No paid calls.**

### LUMI-127 — Extended placeholder resolver

**Description:**
Directive prompts are seeded by live state via `{placeholders}` (`Core.resolve` / `core/placeholders.py`).
Extend the resolver with the new placeholders the tool-thoughts need, **lazy** (resolved at fire time),
`""`-on-empty, and **isolation-aware** (a placeholder never pulls another user's data). (`/core`.)

**What needs to be done:**
- Add resolvers for **`{ambient_news}`** (the v0.4 startup snapshot), **`{world}`** (world context),
  **`{last_image}`** (the most recent sandbox image), **`{interest}`** / **`{hungriest_need}`** (inner-life
  seeds, `""` until v1.1), **`{section}`** (a news section), **`{weekday}`** (from the v0.4 clock), **`{gap}`**
  (the away-gap).
- Each resolver is **lazy** (called only when the placeholder appears), returns **`""` on empty/unavailable**
  (never raises, never blocks), and is **isolation-aware** (scoped to the active user where applicable).
- Unknown placeholders pass through unchanged (the existing behaviour).

**Dependencies:** none new (extends the v0.12 `resolve` + `core/placeholders.py`).

**Expected result:**
A directive topic containing `{weekday}` / `{ambient_news}` / … resolves against live state at fire time;
an unavailable source yields `""` (the prompt degrades, never errors); no placeholder leaks across users.

**Acceptance criteria:**
- [ ] The new placeholders resolve from live state; **`""`-on-empty**; lazy; unknown placeholders pass through.
- [ ] Isolation-aware — a placeholder never returns another user's data (a contract assertion).
- [ ] Unit tests: each placeholder resolves + degrades to `""`; isolation. **No paid calls.**
