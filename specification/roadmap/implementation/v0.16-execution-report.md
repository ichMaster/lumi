### LUMI-063: Index on write + one-time backfill (`core/agent.py`)

**Status:** completed · **Commit:** 204ed8c · **GitHub:** #67

- `Core` takes an optional `embedder` + `recall_enabled`; `_index_messages()` embeds the user +
  Лілі messages right after they're persisted in `reply()` (best-effort — an embed error is logged,
  the turn still completes, the messages are stored). `backfill_vectors(limit)` catches a cold store
  up via `has_vector` (idempotent, bounded); `ensure_backfill()` runs it once lazily.
- `build_core` builds the embedder only when `LUMI_RECALL` is on; a build failure degrades recall to
  off rather than crashing startup.

**Validation:** integration (9) pass · full suite 548 · ruff pass.

### LUMI-064: `/recall <query>` — explicit top-K semantic search

**Status:** completed · **Commit:** 86357f4 · **GitHub:** #68

- `Core.recall(query, k)` — embeds the query, runs `ensure_backfill()`, returns this user's top-K by
  cosine (dated, scored); off / no embedder / empty query / embed error → `[]` (never raises).
- `tui/app.py`: `/recall <query>` renders each hit as `<date> · <who>: «text» (score)`; empty query /
  no match / recall-off → a friendly note; subtitle hint updated. Config `LUMI_RECALL_K` (default 5).

**Validation:** integration + TUI pilot (25 in the run) pass · full suite 558 · ruff pass.
