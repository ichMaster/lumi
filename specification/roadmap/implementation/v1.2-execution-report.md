### LUMI-180: Decouple input from busy + extract `_handle_input_line`

**Status:** completed · **Commit:** d6ac5f8
**Files:** `core/config.py`, `tui/app.py`, `tests/integration/test_tui.py`

- `input_buffer` config + `LUMI_INPUT_BUFFER` (off by default).
- `_set_busy` stops disabling the widget when the buffer is on — `_busy` stays the single-turn model
  mutex the proactive workers respect. Off → `prompt.disabled = busy` (today's behavior).
- Extracted `_handle_input_line(text, *, echoed=False)` (the full command table + `%directive` router
  + chat fall-through) — the one path shared by the live submit and the coming drain.
- `_run_turn` gains `already_shown` (from `echoed`) to suppress the duplicate user row for a queued line.

### LUMI-181: Enqueue + immediate echo while busy

**Status:** completed · **Commit:** 06d3032
**Files:** `tui/app.py`, `tests/integration/test_tui.py`

- A submit while `_busy` (buffer on) → `_enqueue_input(text)`: appends to a FIFO `deque` and echoes the
  full "you sent X" surface **once, at submit** (chat row + Telegram mirror + send sound) for a plain
  chat line; a queued command/`%directive` shows no row (matching the live path). Box clears + keeps
  focus; no turn starts. Off / not busy → today's path.

### LUMI-182: Drain — one turn per queued line, FIFO, never merged

**Status:** completed · **Commit:** f77dc6d
**Files:** `tui/app.py`, `tests/integration/test_tui.py`

- `_run_turn` finally kicks off a guarded `_drain_loop` (a nested kick from the drained turn's own
  finally is absorbed by `_draining`). The loop pops each queued line FIFO and runs it through
  `_handle_input_line(echoed=True)` to completion — a chat line runs a full turn (busy-serialized), a
  queued command runs inline, so the chain **never stalls on a command**. **Each message gets its own
  reply; never merged.** A line typed mid-drain goes to the back.

### LUMI-183: Proactive guards, pending-count status, off-pin
