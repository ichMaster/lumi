## Detailed Results

### LUMI-098: send_image tool + telegram_sink seam

**Status:** completed · **Commit:** 1987976
**Files changed:**
- `core/sendimage.py` (new) — the `send_image(path[, caption])` tool (`SendImageTools`): resolves the
  picture under the per-user sandbox via the shared `safe_path` guard + the v0.22 image-type check, then
  calls the **injected `telegram_sink(abs_path, caption)`**; returns `sent <name> to Telegram` or an error
  string. **Never raises**; on any failure the sink is **not** called. `TelegramSink` seam + `SEND_TOOLS`.
- `core/agent.py` — `telegram_sink` Core/`build_core` param (default `None`); `_sendimage_tool_args`
  registered in `_turn_tools` behind `LUMI_IMAGE`. The core never imports the bridge or writes the outbox.
- `tests/unit/test_sendimage.py` — 9 unit tests (fake sink).

**Validation:** unit 9 pass · contract + image suites green · ruff/compile pass.
