# v0.14 — GitHub Issues

Issues for phase **v0.14 — Local voice (ElevenLabs)** (version **v0 — TUI**), derived from
[ROADMAP.md](../../ROADMAP.md) (§v0.14) and [VOICE_LOCAL.md](../../features/VOICE_LOCAL.md). IDs
continue from v0.13 (LUMI-056 → **LUMI-057…059**).

Hear Лілі — a separate local app that voices her replies in her ElevenLabs voice, no server. The
voicer is the **twin of the v0.13 `outbox→telegram` daemon** (here `outbox → speaker`): it **reuses
the v0.13 file bus** (`outbox.jsonl` + `state/fifo.py`, records `{id, text, emotion, ts, kind}`),
reads new ids via `fifo.read_since` from a **`spoken` pointer**, **voices only `kind="lili"`** (her
replies — your mirrored `kind="user"` keyboard lines are **skipped, never voiced**), one at a time
in ascending order, retries on failure, and **skips the pre-existing backlog on first run**. The
**only** core/TUI change is a one-line outbox gate (write on **voice OR bridge**). ElevenLabs is a
second cloud dependency (`ELEVENLABS_API_KEY` + internet) — **optional/toggle**, `elevenlabs` an
optional dep, **mock TTS** in tests (no paid calls). Depends on v0.13 (the bus), v0.1, v0.3.

## v0.14 — Local voice

### LUMI-057 — Outbox gating (voice OR bridge) + voice config

**Description:**
The voicer reads `outbox.jsonl`, but today the TUI writes it **only** when `LUMI_BRIDGE=on`.
Generalize the gate so voice gets the outbox too, and add the voice config. (`/tui`, `/core`.)

**What needs to be done:**
- **Config** (`core/config.py`): `voice` (`LUMI_VOICE`, off by default), `elevenlabs_api_key`
  (`ELEVENLABS_API_KEY` — a **secret**, never logged/committed), `voice_id` (`LUMI_VOICE_ID`),
  `voice_model` (`LUMI_VOICE_MODEL`, default `eleven_multilingual_v2` for Ukrainian).
- **TUI** (`tui/app.py`): write Лілі's reply to the outbox when **voice OR bridge** is on — a one-line
  change to the existing `mirror_reply` gate in `_run_turn`; set `_outbox_path` in `on_mount` when
  `bridge OR voice`. The keyboard-line mirror (`mirror_user`) **stays bridge-only** (the voicer skips
  `kind="user"` regardless).
- **Test isolation** (`tests/conftest.py`): force `LUMI_VOICE=off` (alongside `LUMI_BRIDGE=off`) so
  no test writes the real outbox via voice.

**Dependencies:** None (builds on the v0.13 outbox).

**Expected result:**
With `LUMI_VOICE=on` (no bridge), Лілі's replies still land in `outbox.jsonl` (`kind="lili"`).

**Acceptance criteria:**
- [ ] `LUMI_VOICE` / `ELEVENLABS_API_KEY` / `LUMI_VOICE_ID` / `LUMI_VOICE_MODEL` config (key never logged).
- [ ] The TUI writes Лілі's reply (`kind="lili"`) to the outbox when **voice OR bridge** is on.
- [ ] `conftest` forces `LUMI_VOICE=off`; tests never touch the real outbox.
- [ ] Unit: config defaults + overrides; the gate condition. No paid calls.

### LUMI-059 — The voicer app (`voice/voicer.py`)

**Description:**
The voicer process — daemon 2's twin — voicing **only Лілі's lines** from the outbox, in order,
exactly once each, resumable. (`/voice`.)

**What needs to be done:**
- `voice/voicer.py`, the **pure, tested** core: `voice_pending(outbox_path, spoken_path, tts, play)
  -> int` — `fifo.read_since` from the `spoken` pointer; for each record ascending: **`kind="user"`
  → skip** (advance the pointer past it, don't voice); **`kind="lili"` → voice** (`play(tts.synth(text,
  emotion=…))`, advance on success); on a synth/play **failure → stop** (leave the pointer before the
  failed id — retry next loop). **First-run skip:** if no `spoken` pointer yet, initialize it to the
  current last outbox id (don't replay the backlog).
- `run()` — the glue (`python -m voice.voicer`, like the telegram daemons): build `ElevenLabsTTS`
  + a local `play`, do the first-run skip, loop `voice_pending` with a small sleep; lazy
  `elevenlabs`; `# pragma: no cover`. The `spoken` pointer = `outbox_path.with_suffix(".spoken")`.

**Dependencies:** LUMI-058 (the TTS adapter).

**Expected result:**
With the voicer running, Лілі's replies are spoken in order, exactly once each; your keyboard lines
are never voiced; the pre-existing outbox is not replayed on first start.

**Acceptance criteria:**
- [ ] Voices `kind="lili"` ascending, one at a time; **`kind="user"` skipped** (pointer still advances).
- [ ] **Retry-on-failure**: a failed synth/play does **not** advance the pointer → re-voiced, never lost/doubled.
- [ ] **First-run skip**: a fresh voicer starts from the current tail (no backlog replay).
- [ ] **Resume**: the `spoken` pointer persists across a restart.
- [ ] Unit tests via `MockTTS` + a mock `play` (no network, no real sleeps, no paid calls).
