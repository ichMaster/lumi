# v0.31 — GitHub Issues

Issues for phase **v0.31 — Semantic recall IV: the recall tool (model-callable memory search)** (version
**v0 — TUI**), derived from [ROADMAP.md](../../ROADMAP.md) (§v0.31) and
[SEMANTIC_RECALL.md](../../features/SEMANTIC_RECALL.md). IDs continue from v0.30 (LUMI-119 → **LUMI-120…122**).

A small **refinement of the recall line** (v0.16 index + v0.17 auto-RAG), **not a new capability** — it
exposes the shipped `recall_moments(query)` (over `repository.search_vectors` + the `Embedder`) as a
**model-callable tool** on the **same v0.19 bounded loop** the file/wiki/news tools use. Auto-RAG (v0.17)
stays the default **"push"** — relevant memory surfaces *unprompted*; the tool adds the **"pull"** for what
auto-RAG can't serve: a **query ≠ the current message** ("а що вони казали про брата?") and **iterative**
search→refine during reasoning.

**The one distinction from the other tools:** a recall result is **her own past = trusted history** (not
untrusted external data — ARCHITECTURE §Semantic recall), so the loop frames it as her **recollection** — the
one tool whose results she treats as her own memory. Results **dedup** against the live window + the v0.17
auto-RAG block (no double-injection); a no-hit / embedder-off degrades to a notice, never blocks the turn.

**Hard rules:** **per-user isolation, unchanged** — the search runs **only over the active user's** vectors
(A↔B contract test); the `{reply, emotion, intensity}` field and the `Embedder`/`VectorStore` seams are
**untouched**; **off by default** (`LUMI_RECALL_TOOL`, needs `LUMI_RECALL` + the embedder) → identical to
v0.17 (auto-RAG only, the tool absent). Also the **engine for the `%recall` directive** (the inward
memory-resurfacing tool-thought, v0.32).

Depends on **v0.16** (the index + the `Embedder`/`VectorStore` seams, `recall`/`search_vectors`), **v0.17**
(the auto-RAG `_recall_block` + dedup + `recall_moments`), and **v0.19** (the bounded tool-loop +
`_turn_tools`) — all shipped. The **embedder is mocked** in every test — no paid calls.

## v0.31 — Semantic recall IV: the recall tool (model-callable memory search)

### LUMI-120 — The `recall` tool + the `LUMI_RECALL_TOOL*` config

**Description:**
Expose the shipped `recall_moments(query)` as a **model-callable tool** on the v0.19 bounded loop, so Лілі can
issue a **targeted** semantic query mid-turn (different from the literal message) and do **multi-hop**
search→refine — the **"pull"** that complements the v0.17 auto-RAG **"push."** Off by default. (`/core`.)

**What needs to be done:**
- A **`recall(query[, k])`** tool (`core/`) over the shipped `recall_moments` — returns the top-`k` relevant
  past moments (snippet + when), **capped**; registered on the v0.19 loop via `_turn_tools` **behind a flag**;
  a **per-turn call cap** (so a turn can't loop forever).
- **Config** (`core/config.py`): `recall_tool` (`LUMI_RECALL_TOOL`, **off**; needs `LUMI_RECALL` + the
  embedder), `recall_tool_k` (`LUMI_RECALL_TOOL_K`), `recall_tool_max_calls` (`LUMI_RECALL_TOOL_MAX_CALLS`);
  the auto-RAG `LUMI_RAG` is unchanged.
- Off → the tool is **not registered** (absent from `_turn_tools`); behaviour identical to v0.17.

**Dependencies:** none new (over the shipped v0.16/v0.17/v0.19).

**Expected result:**
With `LUMI_RECALL_TOOL=on` (+ `LUMI_RECALL` + the embedder), a turn can call `recall("…")` and receive up to
`k` dated moments; the per-turn cap bounds repeated calls. Off → the tool is absent and the turn behaves
exactly like v0.17.

**Acceptance criteria:**
- [ ] `recall(query[, k])` returns the top-`k` `recall_moments` (snippet + when), capped; registered on the v0.19 loop via `_turn_tools` behind `LUMI_RECALL_TOOL`.
- [ ] A per-turn call cap (`LUMI_RECALL_TOOL_MAX_CALLS`) bounds repeated calls within one turn.
- [ ] `LUMI_RECALL_TOOL` (off) + `LUMI_RECALL_TOOL_K` / `_MAX_CALLS` thread through config; on needs `LUMI_RECALL` + the embedder.
- [ ] Off → the tool is absent from `_turn_tools`; a turn behaves identically to v0.17.
- [ ] Unit tests (mock embedder): the tool returns top-`k` moments; the per-turn cap; off → absent. **No paid calls.**

### LUMI-121 — Trusted-recollection framing + dedup vs the window/auto-RAG

**Description:**
Make the tool's results behave as **her own memory**, not external data. The v0.19 loop frames a `recall`
`tool_result` as her **recollection** (distinct from the **untrusted** framing the file/wiki/news tools get —
ARCHITECTURE §Semantic recall), and dedups recall moments against what's already in the prompt (the live
window + the v0.17 auto-RAG block) so nothing is double-injected. The search stays **user-scoped**. (`/core`.)

**What needs to be done:**
- **Trusted framing:** the loop marks a `recall` `tool_result` as **her own recollection** — she treats it as
  memory, not external info; this is the **one** tool whose results are trusted (not the untrusted-data
  framing of wiki/news).
- **Dedup:** recall moments dedup against the **live window** + the **v0.17 auto-RAG block** (reuse the
  LUMI-071/072 `window_ids` dedup) — no double-injection of a moment already in the prompt.
- **Isolation (reused):** the search runs **only over the active user's** store (the v0.16 isolation
  invariant — `recall_moments` is already user-scoped; assert it is not widened here).
- **Graceful:** a no-hit / embedder-off `recall` call returns a short notice (not an error), never blocks the
  turn.

**Dependencies:** LUMI-120.

**Expected result:**
A `recall` result is woven into the reply **as her own memory** (trusted, framed as recollection), never as
untrusted external data; a moment already in the live window or the auto-RAG block is **not** re-injected; a
no-hit / embedder error degrades to a notice; the search never leaves the active user's store.

**Acceptance criteria:**
- [ ] A `recall` `tool_result` is framed **trusted** (her recollection), distinct from the untrusted wiki/news framing.
- [ ] Recall moments dedup against the live window + the v0.17 auto-RAG block (no double-injection).
- [ ] The search runs only over the active user's vectors (isolation reused, not widened).
- [ ] No-hit / embedder-off → a notice, never raises, never blocks the turn.
- [ ] Unit tests (mock embedder): trusted framing on the result; dedup against the window. **No paid calls.**

### LUMI-122 — Contract + isolation + behaviour/degradation tests

**Description:**
Pin the recall-tool surface against the mock embedder (no paid calls): the **trusted-vs-untrusted** framing
contract, the **isolation invariant** (A's `recall` never returns B's moments), the **off-path equivalence**
(off → the tool is absent, behaviour identical to v0.17), an end-to-end **targeted recall** (a query ≠ the
message, used mid-turn), and **graceful degradation**. Extends the contract + isolation test suites.
(`/tests`.)

**What needs to be done:**
- **Contract:** a `recall` `tool_result` is framed **trusted** (her recollection) — distinct from the
  **untrusted** framing the wiki/news tools' results get (a contract test pinning the distinction).
- **Isolation:** user A's `recall` never returns user B's moments (the A↔B test, extended to the recall tool).
- **Off → identical:** with `LUMI_RECALL_TOOL` off, the tool is **absent** from `_turn_tools` and behaviour is
  identical to v0.17 (auto-RAG only).
- **Behaviour end-to-end:** a turn issues a **targeted** `recall` query (**≠** the literal message) and weaves
  the returned moment into the reply.
- **Graceful degradation:** a no-hit / embedder error → a notice, the turn completes, never raises; the
  `{reply, emotion, intensity}` contract validates with the recall tool active.

**Dependencies:** LUMI-120, LUMI-121.

**Expected result:**
`pytest` proves the recall result is **trusted** (distinct from untrusted tools), the search is single-user
(A↔B), the off-path equals v0.17, a targeted recall (≠ the message) is used mid-turn, degradation paths
complete, and the emotion contract holds — all on the mock embedder, no paid calls.

**Acceptance criteria:**
- [ ] Contract: a `recall` result is framed **trusted** (her recollection), distinct from the untrusted wiki/news framing.
- [ ] Isolation: A's `recall` never returns B's moments (A↔B contract).
- [ ] Off (`LUMI_RECALL_TOOL`) → the tool is absent; behaviour identical to v0.17 (auto-RAG only).
- [ ] A turn issues a targeted `recall` query (≠ the message) and uses the moment; a no-hit / embedder error degrades to a notice (never raises); the `{reply, emotion, intensity}` contract validates.
- [ ] All via the **mock embedder** — no paid calls.
