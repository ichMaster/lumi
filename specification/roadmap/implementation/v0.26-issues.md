# v0.26 — GitHub Issues

Issues for phase **v0.26 — Local dictation (STT)** (version **v0 — TUI**), derived from
[ROADMAP.md](../../ROADMAP.md) (§v0.26) and [DICTATION_LOCAL.md](../../features/DICTATION_LOCAL.md). IDs
continue from v0.25 (LUMI-103 → **LUMI-104…106**).

Talk *to* Лілі — a separate local app that **hears your speech and types it into the chat**. The exact
**mirror of the v0.14 voicer**: the voicer reads Лілі's replies from `outbox.jsonl` and **speaks** them;
the dictator **listens to the mic, recognizes Ukrainian, and writes your line into `inbox.jsonl`** — the
same channel the TUI keyboard feeds, so **the core can't tell typed from dictated**. Fully local, a
separate process, no server, no desktop GUI.

It reuses what already ships: the **`state.fifo`** bus (the dictator writes `inbox.jsonl` the same way the
v0.13 Telegram daemon does), the **TUI inbox drain** (`_poll_inbox` / `_drain_inbox` — generalized from
bridge-only to "bridge **or** dictation"), and the **`/voice` adapter pattern** (a thin injected seam, the
twin of `voice/tts.py`'s `TTS`/`MockTTS`). Control is a **TUI toggle key** that flips a shared
**`listen.flag`** (`on`/`off`); the dictator records while `on` and recognizes on `off`. **Toggle, not
hold** (most reliable in a terminal). The web sibling is **v3.4** (same STT adapter).

**Hard rules** (DICTATION_LOCAL.md): the **core is untouched** — a dictated line is an ordinary
`inbox.jsonl` record (`source:"voice"` is reference only); **empty / low-confidence recognition writes
nothing** (better silent than garbage); **dedup by `id`** (the fifo's monotonic counter, like the voicer);
**Ukrainian**, provider configurable (Deepgram Nova-3 uk / ElevenLabs Scribe / offline Whisper) behind one
**STT adapter** (never an SDK in the seam); **off by default** (`LUMI_DICTATION`); recording is **local**
(internet only for a cloud STT call, or run offline Whisper). **No `{reply, emotion, intensity}` change.**

Depends on **v0.1** (the core consumes user turns), **v0.14** (the local-process + shared-file + `/voice`
adapter pattern), **v0.13** (the `inbox.jsonl` fifo bus + the TUI inbox drain) — all shipped. STT is
**mocked** in every test (a `MockSTT` returning canned text — **no audio, no network, no paid calls**).

## v0.26 — Local dictation (STT)

### LUMI-104 — `voice/stt.py` (the STT adapter seam) + `voice/dictator.py` (the dictator process)

**Description:**
Add the **`/voice` STT adapter** (`voice/stt.py`) — a thin injected seam `recognize(audio) -> text`, the
twin of `voice/tts.py`'s `TTS` — and the **dictator** process (`voice/dictator.py`), the mirror of
`voice/voicer.py`: watch `listen.flag`, record the mic while `on`, recognize Ukrainian on `off`, and
append your line to `inbox.jsonl`. Pure helpers fully unit-testable with a `MockSTT`. (`/voice`.)

**What needs to be done:**
- **`STT` seam** (`voice/stt.py`): a `recognize(audio: bytes, *, lang: str = "uk") -> str` Protocol; cloud
  impls behind **lazy** imports (optional deps) — `DeepgramSTT` (Nova-3 uk) and/or `ElevenLabsScribeSTT`,
  plus an **offline** `WhisperSTT` option; a **`MockSTT`** that returns canned text + records calls (no
  audio, no network). **No SDK in the seam's import path** (lazy, like `ElevenLabsTTS`).
- **Dictator pure helpers** (`voice/dictator.py`): `read_flag(path) -> bool` (parse `on`/`off`);
  `recognize_and_append(audio, inbox_path, stt, *, lang) -> int | None` — recognize → **empty /
  low-confidence → return `None`, write nothing**; else `fifo.append(inbox_path, text, source="voice")`
  and return the new id (dedup is the fifo's monotonic id, like the voicer's `spoken` pointer).
- **`run()` glue** (uncovered, like the voicer): watch `listen.flag`; while `on`, record from the mic via
  a sound library (lazy import); on the `on→off` transition, call `recognize_and_append`; reset and wait.
  Requires `LUMI_DICTATION` + the provider key (or offline Whisper).
- **Config** (`core/config.py`): `dictation` (`LUMI_DICTATION`, off), `stt_provider`
  (`LUMI_STT_PROVIDER`: `deepgram`/`whisper`/`elevenlabs`), `stt_lang` (`LUMI_STT_LANG`, default `uk`),
  `listen_flag_path` (`LUMI_LISTEN_FLAG`, default `.lumi/listen.flag`); provider keys
  (`DEEPGRAM_API_KEY` / reuse `ELEVENLABS_API_KEY`).

**Dependencies:** None (mirrors the shipped `voice/tts.py` + `voice/voicer.py` + `state.fifo`; the STT is
an injected seam).

**Expected result:**
`recognize_and_append(b"<audio>", inbox, MockSTT("привіт"))` appends `{id, text:"привіт", source:"voice",
ts}` to `inbox.jsonl` and returns the id; an empty/low-confidence recognition writes nothing and returns
`None`; `read_flag` parses `on`/`off`. No audio, no network.

**Acceptance criteria:**
- [ ] `STT` seam (`recognize(audio, *, lang) -> text`); cloud impls lazy (optional deps) + offline Whisper option + `MockSTT` (canned, records calls); **no SDK in the import path**.
- [ ] `read_flag` (on/off) + `recognize_and_append` (recognize → append `{id, text, source:"voice", ts}` via `fifo`; **empty/low-confidence → no write, returns `None`**; id-dedup via the fifo counter).
- [ ] `run()` glue watches `listen.flag`, records while `on`, recognizes on `off`; gated by `LUMI_DICTATION` + a key/offline.
- [ ] Config `LUMI_DICTATION` / `LUMI_STT_PROVIDER` / `LUMI_STT_LANG` / `LUMI_LISTEN_FLAG` threaded.
- [ ] Unit tests with **`MockSTT`** (no audio/network): append-on-recognize, empty-drop, id-dedup, `read_flag` on/off. **No paid calls.**

### LUMI-105 — TUI listen toggle + `listen.flag` + drain `inbox` for dictation + the recognized line + docs

**Description:**
Wire dictation into the TUI: a **toggle key** flips `listen.flag` (`on`/`off`) and shows a "listening…"
state; the TUI **drains `inbox.jsonl` when dictation is on** (generalizing the bridge-only `_poll_inbox`)
and shows the recognized "you: …" line before submitting it to `core.reply()` — identical to typed input.
(`/tui`.)

**What needs to be done:**
- **The toggle key** (e.g. F2): write `listen.flag = on/off` (the TUI is the **only** writer of the flag);
  show a **"listening…"** indicator while `on`. Off by default (`LUMI_DICTATION`).
- **Drain `inbox` for dictation:** generalize the gate on `_poll_inbox` / `_drain_inbox` from
  `self._bridge` to **`self._bridge or self._dictation`**, and set `self._inbox_path` / `.pos` when
  dictation is on; a dictated line is drained, **shown as your line** (a voice marker), and submitted to
  `core.reply()` exactly like a typed turn. (Works whether or not the Telegram bridge is on.)
- **Config threading** (`cfg.dictation`, `cfg.listen_flag_path`) through `on_mount`.
- **Docs:** `.env.example` (the `LUMI_DICTATION` / `LUMI_STT_*` / `LUMI_LISTEN_FLAG` block) + a
  `docs/DICTATION_SETUP.md` operator guide (the three processes — TUI + dictator (+ voicer); the toggle
  key; providers incl. offline Whisper; the mic permission note).
- **No core change:** the dictated line is an ordinary `inbox` record; `core.reply` is unchanged.

**Dependencies:** LUMI-104.

**Expected result:**
With `LUMI_DICTATION=on` and the dictator running, pressing the listen key shows "listening…" and writes
`listen.flag=on`; speaking a Ukrainian phrase appends a recognized line to `inbox.jsonl`; the TUI drains
it, shows "ти: …" (voice-marked), and Лілі answers — identical to typing it; the bridge need not be on.

**Acceptance criteria:**
- [ ] The toggle key writes `listen.flag` `on`/`off` (TUI = the sole writer) + a "listening…" indicator; off by default.
- [ ] The TUI drains `inbox` when **dictation OR the bridge** is on; a dictated line is shown (voice-marked) and submitted to `core.reply()` identically to a typed turn.
- [ ] Config (`dictation`, `listen_flag_path`) threaded; `.env.example` + `docs/DICTATION_SETUP.md` written.
- [ ] `core.reply` is unchanged (the core can't tell typed from dictated).
- [ ] Integration test (a **fake dictator** writes a line to `inbox`; mock model): the TUI drains + shows + drives a turn identical to a typed one. **No audio/paid calls.**

### LUMI-106 — Integration + contract: recognized line → `inbox` → a turn identical to typed

**Description:**
Pin the whole voice-in path: a recognized line via **`MockSTT`** lands in `inbox.jsonl` and drives a turn
identical to a typed line; empty/low-confidence writes nothing; dedup by id; dictation works independent
of the Telegram bridge; the core is untouched. (`/tests`.)

**What needs to be done:**
- **End-to-end test:** `recognize_and_append` (mock STT) → an `inbox` record → the TUI drain submits it to
  `core.reply()` → a normal `EmotionState`; the turn is **byte-identical** to the same line typed (the
  core sees the same text; `source:"voice"` is reference only).
- **Empty-drop test:** an empty / low-confidence recognition writes **no** `inbox` record and drives **no**
  turn.
- **Dedup test:** the fifo's monotonic `id` prevents a replayed/duplicate line from being doubled.
- **Bridge-independence test:** the inbox drains for dictation with the **Telegram bridge off** (dictation
  and the bridge are independent enablers of the inbox drain).
- **Contract:** the `{reply, emotion, intensity}` contract is unchanged with a dictated turn; the core has
  no dictation-specific code path.

**Dependencies:** LUMI-104, LUMI-105.

**Expected result:**
`pytest` proves a dictated line (mock STT) reaches the core identically to a typed one, empty recognition
is dropped, ids dedup, dictation runs without the bridge, and the emotion contract holds — with no audio,
no network, no paid calls.

**Acceptance criteria:**
- [ ] A recognized line (MockSTT) → `inbox` → a turn identical to a typed line (same `core.reply`, voice-marked in the TUI).
- [ ] Empty / low-confidence recognition → no `inbox` write, no turn.
- [ ] Dedup by the fifo `id` (no doubled lines).
- [ ] Dictation drains `inbox` with the Telegram **bridge off** (independent enablers).
- [ ] The `{reply, emotion, intensity}` contract holds with a dictated turn; the core has no dictation path; STT **mocked** — no audio, no network, no paid calls.
