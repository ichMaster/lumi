# v0.17 — GitHub Issues

Issues for phase **v0.17 — Semantic recall II: automatic RAG in the turn** (version **v0 — TUI**),
derived from [ROADMAP.md](../../ROADMAP.md) (§v0.17),
[SEMANTIC_RECALL.md](../../features/SEMANTIC_RECALL.md) §v0.17, and
[SEMANTIC_RECALL_CONTEXT.md](../../features/SEMANTIC_RECALL_CONTEXT.md). IDs continue from v0.15
(LUMI-069 → **LUMI-070…074**).

v0.16 built the foundation — every message is embedded into a per-user vector store, searchable on
demand via `/recall`. v0.17 makes recall **automatic**: each turn, the incoming message is the query,
the most relevant past moments are retrieved and **injected into the reply** — so Лілі remembers the
exact thing you said long ago, right when the topic returns. And per
[SEMANTIC_RECALL_CONTEXT.md](../../features/SEMANTIC_RECALL_CONTEXT.md), a hit is injected **with its
neighbours** (a dated dialogue snippet), not as an orphan line.

The block lives in the **volatile tail** of the prompt (per-turn → never cached — `cache-the-static /
RAG-the-dynamic`, [PROMPT_OPTIMIZATION.md](../../../docs/PROMPT_OPTIMIZATION.md) §3), labelled as
**recall** (trusted history, distinct from untrusted web content). Best-effort and **non-blocking**
(like ambient context): error/empty → no block, never delays a turn. Reuses the v0.16 seams unchanged
(`Embedder`, `VectorStore`); **same per-user isolation invariant** — retrieval (and the neighbour
window) runs only over the requesting user's data.

Depends on **v0.16** (LUMI-061…065 — the index + seams, all shipped). No core **contract** change to
the emotion field `{reply, emotion, intensity}`.

## v0.17 — Semantic recall II

### LUMI-070 — Per-turn retrieval + the «Релевантні моменти минулого» block

**Description:**
Make recall automatic: each turn, embed the incoming user message and inject the most relevant past
lines into the reply. The retrieval + the prompt block + the non-blocking wiring — the bounds come in
LUMI-071, the neighbour-window in LUMI-072. (`/core`.)

**What needs to be done:**
- **Retrieval** (`core/agent.py`): in `Core.reply`, before the model call, embed the incoming message
  (`embed([text], is_query=True)`) and `search_vectors(self._user_id, q, k)` (default `LUMI_RAG_K`),
  applying a **relevance floor** (drop hits below `LUMI_RAG_FLOOR`). Reuses the v0.16 embedder + store.
- **Block assembly:** render the surviving hits into a dated block — `# Релевантні моменти минулого`
  with one dated line per hit (`— YYYY-MM-DD · who: «…»`), newest/most-relevant first.
- **Inject into the tail** (`core/prompt.py`): `build_system_prompt` gains a `recall: str | None`
  param appended to the **per-turn tail** (never the cached prefix — it changes every turn), labelled
  as recall so the model treats it as a remembered fragment, not current flow. The
  `system.startswith(cache_prefix)` invariant holds (the block is tail-only).
- **Best-effort + non-blocking:** wrap retrieval in try/except → on error/empty inject **no block**
  and never delay the turn (degrades like ambient context). Off (`LUMI_RAG=off`) / no embedder →
  no-op. Runs `ensure_backfill` first so a cold store still answers (consistent with `/recall`).
- **Config** (`core/config.py`): `rag` (`LUMI_RAG`, **off** by default), `rag_k` (`LUMI_RAG_K`,
  default e.g. 4), `rag_floor` (`LUMI_RAG_FLOOR`, a cosine threshold e.g. 0.3); threaded `build_core`
  → `Core`.

**Dependencies:** None (builds on v0.16 — shipped).

**Expected result:**
You mention пуер months later; the turn's prompt carries a `# Релевантні моменти минулого` block with
the old пуер lines, and Лілі answers grounded in them — automatically, no `/recall` typed.

**Acceptance criteria:**
- [ ] `Core.reply` embeds the incoming message and retrieves top-K (relevance floor) over **this user's** vectors.
- [ ] The dated `# Релевантні моменти минулого` block is injected into the **tail** (never the cached prefix); the `startswith(cache_prefix)` invariant holds.
- [ ] Best-effort + non-blocking: error/empty/off/no-embedder → no block, the turn is never delayed or broken.
- [ ] Config `LUMI_RAG` (off), `LUMI_RAG_K`, `LUMI_RAG_FLOOR`; threaded to `Core`.
- [ ] Unit tests via `MockEmbedder` + store: block injected above the floor, empty below it, graceful error. No paid calls.

### LUMI-071 — Dedup against the window + bounds (floor, count + token cap)

**Description:**
Spend recall tokens carefully: never inject a line already in the live window (no double-context),
and cap the block by count and a token/char budget. The relevance floor (LUMI-070) stays the gate.
(`/core`.)

**What needs to be done:**
- **Dedup against the rolling window:** drop any retrieved hit whose message is already in the
  verbatim live tail this turn (compare by the content-addressed `msg_id` / `(ts, role, text)`), so
  the block never repeats current-context lines.
- **Cap:** bound the block by **count** (top-K already) **and** a **token/char budget**
  (`LUMI_RAG_MAX_CHARS`); if hits overflow the budget, keep the most relevant and drop the rest.
- **Floor honored end-to-end:** a turn with no hit above `LUMI_RAG_FLOOR` injects **nothing**
  (zero-cost block) — recall never *adds* a turn's worth of weak noise.
- **Config:** `rag_max_chars` (`LUMI_RAG_MAX_CHARS`, the block's token/char budget).

**Dependencies:** LUMI-070.

**Expected result:**
The recall block never duplicates a line you can already see in the window, and stays within a small,
bounded size; a weak-match turn injects an empty block.

**Acceptance criteria:**
- [ ] Retrieved hits already in the live window are dropped (no double-context).
- [ ] The block is capped by count **and** a token/char budget; overflow keeps the most relevant.
- [ ] Below the floor → empty block (zero cost).
- [ ] `LUMI_RAG_MAX_CHARS` config.
- [ ] Unit tests: dedup vs a seeded window, cap/floor honored, deterministic order. No paid calls.

### LUMI-072 — Context expansion: inject the hit **with its neighbours**

**Description:**
A bare matched line is torn from its thread. Inject each hit **plus a small window of its session
neighbours** as a dated dialogue snippet (the **anchor marked**), so the model grounds in a coherent
moment, not an orphan line — per [SEMANTIC_RECALL_CONTEXT.md](../../features/SEMANTIC_RECALL_CONTEXT.md).
(`/core`, `/state`.)

**What needs to be done:**
- **Resolve a hit to its session position.** Recommended (no re-index): build an in-memory
  `msg_id → (session_id, index)` map for **this user** (compute `vector_msg_id` over the user's stored
  messages once, lazily/cached) so a hit's `msg_id` resolves to its place in its session. (Alternative
  per the spec: add `session_id` to `VectorRecord` — additive, but forces a re-index via the
  `vectors_model` marker; the in-memory map avoids that.)
- **Neighbour window** (`Repository`/`core`): for each hit, take the `W` messages **before and after**
  the anchor **within the same session** (default `LUMI_RAG_W = 2`), in order — `neighbours(session_id,
  index, w)` over `load_messages` (already ordered, per-user).
- **Assemble the snippet:** one dated dialogue block per moment (`— YYYY-MM-DD, <part of day> —` then
  the windowed lines), the **anchor line marked** (`← (matched)`); replaces the bare line from LUMI-070.
- **Merge overlapping windows:** two hits a few turns apart share neighbours → emit **one** merged
  snippet, never the same line twice.
- **Dedup the whole snippet** against the live window (extends LUMI-071 — dedup the expanded snippet,
  not just the anchor); the **count budget funds fewer hits at more context** (`K` down, `W` up).
- **Graceful:** a failed neighbour lookup degrades to the **bare anchor line** (still useful), never
  blocks the turn.
- **Config:** `rag_w` (`LUMI_RAG_W`, default 2).

**Dependencies:** LUMI-070, LUMI-071.

**Expected result:**
`[2026-04-02 03:14] Ти: третю ніч не сплю` resurfaces as a 3-line dated snippet (the line + the turn
before and after, anchor marked), so Лілі sees the thread, not a fragment.

**Acceptance criteria:**
- [ ] A hit resolves to its session position (in-memory `msg_id → (session_id, index)` map — no re-index).
- [ ] Each hit injects a dated dialogue snippet of `±W` neighbours **within the same session**, anchor marked.
- [ ] Overlapping windows merge (no line twice); the snippet is deduped against the live window.
- [ ] A failed neighbour lookup degrades to the bare anchor; never blocks the turn.
- [ ] `LUMI_RAG_W` config. Unit tests via `MockEmbedder` + store: window, merge, anchor mark, degrade. No paid calls.

### LUMI-074 — Per-turn RAG isolation **contract test** + dedup/cap/floor/graceful

**Description:**
Pin the hard rule for the automatic path: per-turn RAG (and its neighbour window) runs **only** over
the requesting user's data — a hit, and the surrounding lines pulled with it, can **never** come from
another user. Plus the bound/graceful guarantees as contract checks. (`/tests`.)

**What needs to be done:**
- Extend `tests/contract/test_semantic_recall_isolation.py` (v0.16): seed users A and B; run a **full
  turn** for B and assert the injected `# Релевантні моменти минулого` block (and every neighbour line
  in it) contains **only** B's messages — never A's, across retrieval **and** the expanded window.
- **Bounds contract:** dedup against the window (no double-context); the count + token cap; the
  relevance floor (below it → empty block).
- **Graceful contract:** a retrieval/embedder/neighbour error → the turn still completes with **no
  block** and is never delayed; `LUMI_RAG=off` → the prompt is byte-for-byte the pre-RAG behaviour.
- Reuse `MockEmbedder` + a fresh per-user store; **no network, no paid calls**.

**Dependencies:** LUMI-071, LUMI-072, LUMI-073.

**Expected result:**
A contract module proves the per-turn RAG never crosses users (anchor *or* neighbours) and degrades
safely — failing loudly if a future change lets a turn's recall leak across users.

**Acceptance criteria:**
- [ ] Contract: the per-turn block (anchor + neighbours) returns **only** the requesting user's messages (A↔B).
- [ ] Contract: dedup vs the window; count + token cap; floor → empty block.
- [ ] Contract: retrieval/neighbour error → no block, turn unbroken; `LUMI_RAG=off` → pre-RAG prompt unchanged.
- [ ] All via `MockEmbedder` (no network, no paid calls); green in CI.

## Definition of Done (phase)

Mirrors [ROADMAP.md §v0.17](../../ROADMAP.md): each turn injects the **most relevant past messages**
(when above the floor) — **with their neighbour-window context** (the moment, not the line) — deduped
against the live window and capped by count + token budget; an old, relevant line **resurfaces when
the topic returns**; retrieval **never blocks a turn or crosses users** (anchor or neighbours);
**off/degraded → behaves like today**. `/recall` reuses the same expansion. The recall block lives in
the **volatile tail** (never cached); **no core contract change** (the emotion field is untouched; the
`Embedder`/`VectorStore` seams are unchanged). The embedder/store are **mocked in tests** (no paid
calls). Builds on v0.16 (the index + seams).
