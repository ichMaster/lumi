# v0.4 — GitHub Issues

Issues for phase **v0.4 — Ambient context (now, here, weather, news)** (version **v0 — TUI**), derived from the per-phase Tasks list in [ROADMAP.md](../../ROADMAP.md) (§v0.4) and the contracts in [ARCHITECTURE.md](../../ARCHITECTURE.md) (§Sessions and history, §Configuration, §Mood and temperament — which this ambient layer feeds). This file is scoped to a single phase; IDs continue from v0.3 (LUMI-018 → **LUMI-019…022**).

v0.4 gives Лілі a sense of **when and where she is**, and lets her **break a long silence first**. A lightweight **ambient context** is fetched **once at startup** (not MCP, not per-turn) — date/time (injected clock), calendar, location, weather, short news — and injected into the system prompt as a "now / here" block: **ambient color, never competence**, fetched text **data not instructions**, each source **config-gated** and degrading gracefully. Two prompt-assembly changes make time legible: **every message carries its date-time** and **short summaries carry their date**. Finally, an **idle nudge** quietly feeds Лілі a user-style opener from an authored list when you've gone quiet — you never see the nudge, only her reply — so she seems to speak first (simple presence-only proactivity; the v4.2 open-loop proactive-turn mechanism generalizes it later).

This phase extends the v0.3 seams without breaking them: `build_system_prompt` gains an opt-in `ambient=…` block (it was authored as an extension seam); `Core.reply` keeps its signature but threads the snapshot, timestamps messages, and dates summaries; the TUI triggers the startup `fetch()` and the idle timer. The injected clock (the same deterministic pattern v0.6 mood needs) keeps everything testable.

## v0.4 — Ambient context

### LUMI-019 — Time-legible prompts — per-message timestamps + dated summaries

**Description:**
Make time legible in the prompt so Лілі perceives the *rhythm* of the conversation and places past sessions in time: every message in the model's `messages` array is prefixed with its date-time, and every injected `ShortSummary` carries its date. (`/core` — prompt assembly + the turn.)

**What needs to be done:**
- In `Core.reply`, prefix each replayed message's content with a **compact local date-time** derived from its `ts` (e.g. `[2026-06-07 14:30] …`). The current user line uses **now** from the injected clock. Keep the v0.3 emotion-tag reconstruction on Лілі's turns (timestamp + `<emotion>` tag both present).
- In `build_system_prompt`, the **summaries block** shows each summary with its date (the summary's `ts`) — e.g. `- [2026-06-05] <summary>`.
- A small deterministic formatter (`ts` → compact local date-time string), unit-tested; no wall-clock reads in the formatter.

**Dependencies:** None.

**Expected result:**
Every message and every recalled summary in the prompt carries a date/time, so the model can tell "just now" from "last week".

**Acceptance criteria:**
- [ ] Each message in the `messages` array is prefixed with its date-time (from its `ts`; the new user line uses the injected-clock "now").
- [ ] Each injected `ShortSummary` carries its date.
- [ ] The date-time format is deterministic (derived from `ts`), unit-tested.
- [ ] The v0.3 emotion-tag reconstruction and the rolling-window/compaction behavior still hold.

### LUMI-020 — `WorldContext` provider — clock + config-gated weather/news/location

**Description:**
The ambient **data layer**: a `WorldContext` provider at the core boundary (interface-independent) whose `fetch()` returns a `WorldContext{ now, calendar, location, weather, news }` snapshot. Time/date come from the **injected clock**; location/weather/news from **thin HTTP** calls to configured providers (no SDK, no tool loop). **Config-gated** and **graceful**: any source off/erroring → that field is `None`, never an exception. Fetched text is **data** (truncated/sanitized), never instructions. (`/core`.)

**What needs to be done:**
- A `WorldContext` dataclass `{ now, calendar, location, weather, news }` (each field optional/`None`).
- A provider with `fetch(clock) -> WorldContext`: `now`/`calendar` (weekday, date, notable dates) computed from the **injected clock**; `location`/`weather`/`news` via thin HTTP (`httpx`/`requests`) to configured endpoints, each wrapped so a timeout/error/empty config yields `None`.
- **Config-gated sources** (off unless configured): `LUMI_LOCATION` (or IP geolocation), weather key + endpoint, news key/RSS + a **headline cap** (3–5, truncated). Clock/calendar need no key. Document each in `.env.example`.
- Treat fetched text as **untrusted data** — truncate headlines, strip control content; never executed/followed.

**Dependencies:** None.

**Expected result:**
A `WorldContext` snapshot (real or mocked); when a source is off or fails, its field is `None` and the rest still populate.

**Acceptance criteria:**
- [ ] `WorldContext{ now, calendar, location, weather, news }`; `now`/`calendar` come from the injected clock (deterministic).
- [ ] Weather/news/location fetched via thin HTTP, **config-gated**; absent config → field `None`.
- [ ] Any source error/timeout → that field `None`; `fetch()` **never raises**.
- [ ] Tests use **mocked HTTP** (no live calls in CI), including each source failing → `None`; the news headline cap is enforced.

### LUMI-021 — Ambient block in the system prompt + startup fetch/refresh

**Description:**
Inject the ambient **"now / here" block** into the system prompt and wire the snapshot through the turn. `build_system_prompt` gains an opt-in `ambient=…`; the `Core` threads the `WorldContext` snapshot; the TUI fetches it **at startup** and re-fetches on `/new` (or a coarse interval). The **date-time recomputes per turn** from the clock, while weather/news/location are the startup snapshot. **Coloring tone, never competence.** (`/core` + `/tui`.)

**What needs to be done:**
- `build_system_prompt(…, ambient=…)`: a short now/here line — date-time, weekday, location, weather, 3–5 headlines — placed **near the canon**. With no ambient → no block (canon stays verbatim; the v0.1 behavior holds).
- `Core` holds the `WorldContext` snapshot; `_system_prompt` assembles the ambient line (date-time **recomputed per turn** from the clock; weather/news/location read from the held snapshot). Fetched text quoted as **data**, never followed.
- The TUI calls the provider `fetch()` **at startup** and passes the snapshot to the `Core`; re-fetch on `/new` (and optionally a coarse interval). A failed/empty snapshot → no block, no error.

**Dependencies:** LUMI-020 (the `WorldContext` provider).

**Expected result:**
Turns carry the ambient block; "now" stays current across the session; an unconfigured/failed run simply has no block.

**Acceptance criteria:**
- [ ] `build_system_prompt(ambient=…)` injects the now/here block; off → canon verbatim (no ambient text).
- [ ] `Core` threads the snapshot; the date-time is recomputed per turn; weather/news come from the startup snapshot.
- [ ] The TUI fetches at startup and refreshes on `/new`; an empty/failed snapshot yields no block and no error.
- [ ] **Integration test:** a turn's system prompt carries the ambient block from a mocked snapshot; an unconfigured run has none.

### LUMI-022 — Idle nudge — a hidden self-started turn

**Description:**
Let Лілі break a long silence. An **idle timer** (driven by the injected clock) fires when the chat is open and there's been no activity for a configurable interval (default ~3–5 min); it picks a line from an authored **nudge list** and runs it through the **normal turn as a hidden user message** — Лілі's reply is shown (with her emotion), the injected user line is **never displayed** — so she appears to speak first. **Config-gated, off by default, rate-limited, quiet-hours-aware.** (`/tui` + a small `/core` hook.)

**What needs to be done:**
- An **idle timer** in the TUI: the decision ("has it been ≥ interval since the last activity?") is a **pure, clock-driven function** (testable, no real sleeps); a Textual timer polls it. Any real user input **resets** it.
- An authored **nudge list** (editable file, e.g. `core/nudges.md`, like the canon/styles): short, user-style openers (e.g. «ти тут?», «про що думаєш?»); pick rotating/random.
- Run the nudge through `Core.reply` as a **hidden user turn**: Лілі's reply renders normally (emotion + ambient + memory); the injected user line is **suppressed** from the on-screen chat, the `transcript`, and one-key copy, and marked `auto-nudge` so it never surfaces as something you typed. (Whether the hidden line persists to history vs. stays ephemeral is a flagged choice — note it in code.)
- **Config**: on/off (**default off**), interval, the nudge file, quiet hours; **rate-limit** (at most one nudge per idle gap). Document in `.env.example`.

**Dependencies:** None (composes with the ambient turn — LUMI-021 — when present).

**Expected result:**
With the nudge on, a long silence (chat open) yields a reply from Лілі and **no visible user message**; real input always resets the timer; off by default.

**Acceptance criteria:**
- [ ] The idle decision is a pure clock-driven function; it fires after the configured gap and is **rate-limited**; any real input resets it — tested **deterministically (no real sleeps)**.
- [ ] The nudge runs as a **hidden user turn** from the authored list; Лілі's reply shows; the injected user line is **absent** from the displayed transcript and the copy/`/forget`-style exports.
- [ ] **Config-gated, off by default**, quiet hours respected.
- [ ] **Integration test:** an idle nudge produces a Лілі reply while the hidden nudge line is not in the displayed transcript.
