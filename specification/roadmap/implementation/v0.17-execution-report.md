## Notes

- **The recall block lives in the volatile tail** — never the cached prefix, so v0.15 prompt caching
  keeps working (`cache-the-static / RAG-the-dynamic`).
- **No re-index needed.** Context expansion resolves a hit to its session position via an in-memory
  map, so the user's existing (Voyage) vectors stay valid — switching v0.17 on requires no re-embed.
- **Off by default.** `LUMI_RAG=off` → behaves exactly like v0.16 (index + `/recall` only). Turn it
  on with `LUMI_RAG=on`. `.env.example` documents `LUMI_RAG` / `_K` / `_FLOOR` / `_W` / `_MAX_CHARS`.
- **Same isolation invariant.** Retrieval and the neighbour window run only over the requesting
  user's data, pinned by the contract.
