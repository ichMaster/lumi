# Version v0.38 — Issue Breakdown

**Phase:** v0.38 — Inner Voice: the authored three-voice think-phase instruction

Лілі's pre-reply reasoning runs today on a generic **hardcoded** directive (`REASONING_DIRECTIVE` in
`core/prompt.py` — "wrap reasoning in `<think>`"). This phase moves it into an **editable
`core/inner_voice.md`** that makes the think-step sound like **her** — a short **three-voice negotiation**
(Імпульс / Тверезість / Стандарт) weighing her **mood** (v0.6/0.8) and **closeness** (v0.10) before she
speaks — plus a **show/log policy** for the monologue. **No new engine** (reuses the shipped think-phase:
Opus extended thinking / the v0.37 gpt-5.5 Responses reasoning + the `<think>` parse + the Thinking box +
`thinking_summary`); only the *instruction* changes, and it becomes a file you edit. **No contract change**;
off → byte-identical. Design: [INNER_VOICE.md](../../features/INNER_VOICE.md) (target state:
[try-holosy-lili.md](../../features/ukrainian/personality/try-holosy-lili.md)).

### LUMI-149 — Inner Voice: authored `core/inner_voice.md` + loader + `LUMI_INNER_VOICE`

**Description:**
Move the hardcoded `REASONING_DIRECTIVE` (`core/prompt.py`) into an **editable** `core/inner_voice.md`
authored as Лілі's **three-voice negotiation** (Імпульс weighs warmth → Тверезість checks the facts + her
states + whether a feeling has an anchor → Стандарт holds the boundaries → **one** weighed reply), weighing
the **mood + closeness** blocks already in the prompt. Behind a `LUMI_INNER_VOICE` toggle; off →
byte-identical to today.

**What needs to be done:**
- **Author `core/inner_voice.md`** (Ukrainian, her voice) — the three voices, the negotiation pass, the
  **feeling-anchor rule** (presence of a link, not quality of emotion), the **five invariants**
  (roles-not-psyche / never-spoken-aloud / Standard-is-support / **never competence** / one-reply-out), and
  the **mechanical `<think>` wrap line** so `split_reasoning` keeps working. A ready draft is in the
  [INNER_VOICE.md](../../features/INNER_VOICE.md) appendix. No deferred-seam comments in the file (they'd
  enter the prompt).
- **Loader** — read the file at startup (path `LUMI_INNER_VOICE_FILE`, default `core/inner_voice.md`).
- **Wire it into the system prompt** — in the `_system_prompt` assembly, when `LUMI_INNER_VOICE=on`, use the
  loaded inner voice **instead of** `REASONING_DIRECTIVE` (the mood/closeness blocks already ride in the
  prompt — it *consumes* them). Off → `REASONING_DIRECTIVE`, byte-identical.
- **Config:** `LUMI_INNER_VOICE` (off by default), `LUMI_INNER_VOICE_FILE`.

**Dependencies:** none new — reuses v0.3 (the emotion turn), v0.6/0.8 (mood — the tone input), v0.10
(closeness — a weighed input), v0.37 (the think infra it reuses). Provider-agnostic (Opus + gpt-5.5).

**Expected result:**
With `LUMI_INNER_VOICE=on`, the reply turn's think-block reads as **her three-voice weighing of her states**
(mood / closeness / the subtext), on both Opus and gpt-5.5; off → today's `REASONING_DIRECTIVE` unchanged.

**Acceptance criteria:**
- [ ] `core/inner_voice.md` is authored, loaded, and **replaces `REASONING_DIRECTIVE`** in the system prompt when on; **off → byte-identical**.
- [ ] A **voice test** (mocked): the think-block references her states (mood/closeness), not generic task analysis; the voices are **never spoken** in the visible reply.
- [ ] **One model call** per reply (housekeeping thinking-off); the five invariants hold inside `<think>`.
- [ ] **No contract change** (`{reply, emotion, intensity}`); **no paid calls** (model mocked).

### LUMI-150 — `LUMI_THINK_SHOW`: think-show modes + the think-log tier

**Description:**
The **show / log policy** for the monologue: a `LUMI_THINK_SHOW` mode controlling visibility, plus a log
tier — the monologue is **logged but never persisted** to long-term memory (it is ephemeral; only a later
v1.4 impression persists).

**What needs to be done:**
- **`LUMI_THINK_SHOW`** — `debug` (operator-visible in the Thinking box, **never** in the reply; safe
  default) / `open` (surfaced as her inner voice — then it MUST stay in character) / `off` (hidden entirely).
  The TUI honours the mode (off → empty box).
- **Think-log tier** — the monologue is appended to the v0.3 logged tier; assert it is **never** written to
  long-term memory (a contract-style memory test pins this).
- **Docs:** note the mode + the never-persisted guarantee.

**Dependencies:** **soft on LUMI-149** (the authored inner voice gives a monologue worth surfacing);
mechanically independent (works on any think-box content).

**Expected result:**
`LUMI_THINK_SHOW` controls whether/how the monologue shows; the raw monologue is logged but never persisted.

**Acceptance criteria:**
- [ ] `think_show=off` hides the box; `debug` shows it to the operator (never in the reply); `open` surfaces it in character.
- [ ] The monologue is **logged**; a **memory test** pins that the raw think is **not persisted** to long-term memory.
- [ ] **No contract change**; **no paid calls**.
