# v0.12 — GitHub Issues

Issues for phase **v0.12 — Thought-stream** (version **v0 — TUI**), derived from the per-phase
Tasks in [ROADMAP.md](../../ROADMAP.md) (§v0.12) and [THOUGHT_STREAM.md](../../features/THOUGHT_STREAM.md)
(+ ARCHITECTURE §Prompt placeholders). IDs continue from v0.11 (LUMI-045 → **LUMI-046…052**).

Лілі's mind **acts on its own**: between and around your messages she **`%think`s** (and `%wonder`s)
from her live state into a private, **dated diary** (the thought-stream), **mostly silently**, and
only **occasionally** surfaces one aloud. A new three-layer vocabulary — **`%directives`** (her mind
acts, internal/never typed) vs **`/commands`** (read a state) vs plain chat she speaks — over **one
reusable "mental-act" engine** (`trigger → seed → generate → record → maybe surface`). Built on the
v0.4 **nudge** (trigger + the hidden self-turn delivery), v0.6 **mood** (the housekeeping-call
pattern + a seed), v0.10 **closeness** (a seed), v0.2 **Repository** (the store) — all shipped. It
launches **thin** (mood + closeness + recent) and **enriches** as v0.13–14 add needs/plans/dreams to
the seed. **No emotion-contract change.** **Global to Лілі** (not per-user; surfacing is
per-conversation — a thought from user A never reaches B). Mockable end-to-end (mock model + fixed
clock + injected seed; no paid calls).

## v0.12 — Thought-stream

### LUMI-046 — Global `Thought` store (dated diary) + isolation contract test

**Description:**
A new **global** store for her thoughts, behind the `Repository` — **not** `user_id`-keyed (like
`InnerLife`/`Needs`). Every record is **date-time stamped** (the injected clock) and appended in
order, so the stream reads as a **dated diary**. (`/state`, `/core`.)

**What needs to be done:**
- `Thought{when, kind, text, emotion, seeds, spoken, ts}` record (`core/repository.py`): `when` =
  the **local diary stamp** (injected clock); `kind` ∈ `{think, wonder, …}`; `text` (her voice);
  `emotion` (the locked base-9 enum); `seeds` (which states fed it); `spoken` (did it graduate);
  `ts` (write-time, audit).
- `Repository` methods + `state/local_store.py` impl: append a thought; read thoughts **since** a
  timestamp / within an age window; a rolling **soft cap by age** (`D` days, config) that fades old
  entries (their gist consolidates into v0.16 impressions later — not pinned here).
- The store is **global** (a single list, not a dict-by-`user_id`). Each thought carries the
  originating `user_id` **only** for the surfacing filter (not for storage scoping).
- **Isolation seam:** a `thoughts_for(user_id, since)` read that returns only thoughts **surfaceable
  to that user** (its own + any de-identified/global ones) — never a thought sparked in another
  user's session.

**Dependencies:** None (builds on v0.2 the `Repository`).

**Expected result:**
Thoughts persist in a global, date-ordered diary; a per-user read never returns another user's
thought.

**Acceptance criteria:**
- [ ] `Thought` record + `Repository`/`local_store` append + age-windowed read; soft cap by age.
- [ ] Store is **global** (not `user_id`-keyed); thoughts carry `user_id` for surfacing only.
- [ ] **Contract test:** a thought written under user A is **never** returned by `thoughts_for(B)` (A→B isolation); the shape is pinned; the store is global, not per-user.
- [ ] Unit tests with a temp-file repo + **fixed clock** (dated ordering, age window, isolation). No paid calls.

### LUMI-047 — Mental-act engine + registry + `%think` / `%wonder`

**Description:**
One reusable **mental-act engine** — `trigger → seed her state → generate (one housekeeping call,
thinking-OFF) → record → maybe surface` — and a small **registry** of directives. Ship `%think`
(everyday musing) and `%wonder` (a curiosity), both writing to the thought-stream. (`/core`.)

**What needs to be done:**
- A directive **registry**: `{name, trigger, seeds, store, surface}` entries; register `%think` and
  `%wonder`. (`%dream`/`%reflect`/`%recall` are **not** built here — they're v0.14/0.16/0.18
  retrofits on the same engine.)
- `thought_request(seeds, *, topic=None, rng_seed)` — the prompt-builder: seeds from **mood**
  (v0.6/0.8), **closeness** (v0.10), the **recent conversation tail**, the **last thoughts**
  (continuity), an optional **topic**, and an **injected random seed**. (Needs/plan join the seed
  when v0.13–14 land — no rework.)
- Generate via the **`LLMClient` seam** (one call, **thinking-off**, like the mood/summary
  housekeeping); parse a structured thought (`text`, `emotion` validated to the base-9 enum,
  `seeds`); **drop malformed** output (generate nothing rather than corrupt the stream).
- Record a `Thought` (LUMI-046) with the directive `kind` and the diary stamp.

**Dependencies:** LUMI-046 (the store).

**Expected result:**
Calling the `%think`/`%wonder` directive produces a structured, in-her-voice thought recorded to the
dated diary; malformed model output is dropped.

**Acceptance criteria:**
- [ ] The engine (`trigger → seed → generate → record → maybe surface`) + a registry with `%think` and `%wonder`.
- [ ] `thought_request` seeds from mood/closeness/recent/last-thoughts (+ optional topic + injected seed); one **thinking-off** call via the mock model.
- [ ] Validation: `emotion` clamped to the base-9 enum; malformed/empty output **dropped** (nothing recorded).
- [ ] Unit tests (mock model + fixed clock + injected seed) — a `%think` records a structured thought; a `%wonder` records with `kind="wonder"`; a malformed reply records nothing. No paid calls.

### LUMI-048 — `{name}` placeholder resolver (registry + isolation-aware)

**Description:**
A small, **fixed-registry** placeholder resolver (ARCHITECTURE §Prompt placeholders): one seam
expands `{name}` tokens to live state at prompt-build time, so **authored** text decides placement.
Reused by authored prompts (canon, `core/inner_voice.md` later, `thought_request`, `nudges.md`) and
**directive topics** (`%think about {last_thought}`). (`/core`.)

**What needs to be done:**
- `resolve_placeholders(text, ctx)` — substitute the **fixed registry**: `{last_thought}`,
  `{thoughts}` (the last-24h dated diary block), `{mood}`, `{closeness}`, `{plan}`, `{need}`,
  `{recent}`, `{now}`/`{today}`, `{user}`. (`{plan}`/`{need}` resolve to empty until v0.13.)
- **Unknown `{token}` → left literal** (and logged) — never an error (so a literal `{...}` or a
  future token survives; a literal `<think>` instruction is irrelevant under `{}`).
- **Isolation-aware:** `{last_thought}`/`{thoughts}` resolve via `thoughts_for(user_id)` (LUMI-046)
  — a token never resolves to another user's thought.
- **Deterministic:** a pure function of `(injected clock, repository, user)` — one seam, not
  per-call string-building.

**Dependencies:** LUMI-046 (`{last_thought}`/`{thoughts}` read the store).

**Expected result:**
Authored strings and directive topics with `{name}` tokens expand to current state; unknown tokens
pass through verbatim; per-user tokens respect isolation.

**Acceptance criteria:**
- [ ] Resolver expands the documented registry; `{last_thought}`/`{thoughts}` from the store, `{mood}`/`{closeness}`/`{recent}`/`{now}`/`{user}` from state/clock.
- [ ] Unknown `{token}` left **literal** (+ logged); never raises.
- [ ] **Isolation:** `{last_thought}` for B never resolves to A's thought (asserted).
- [ ] Unit tests (fixed clock + temp repo) — each token resolves; unknown is literal; isolation holds. No paid calls.

### LUMI-049 — Feedback block — the last-24h dated diary in the prompt + mood nudge

**Description:**
The reason a silent thought is worth a call: it **feeds back**. Inject the **last-24h dated** slice
of the diary into the system prompt as a compact block, and let a recurring thought softly nudge the
mood. (`/core`.)

**What needs to be done:**
- A `# Що в мене на думці (за останню добу)` block in `build_system_prompt` (the same overlay slot
  as the mood/closeness blocks): every thought from the **last `LUMI_THOUGHTS_WINDOW_H` hours**
  (default 24, rolling from the injected clock), **each shown with its time**, **per-user-filtered**
  (LUMI-046). A **hard token/line cap** backstops a busy day; older entries are excluded (they stay
  in the diary).
- The **soft nudge to mood:** a recurring thought feeds the v0.6 mood inputs (tone only, **never
  competence**; the once/day cache + resolution rules carry over). (The needs nudge lands with
  v0.13.)
- Degrade to **no block** when the window is empty (never blocks a turn).

**Dependencies:** LUMI-046 (the store), LUMI-047 (thoughts exist).

**Expected result:**
Each turn she "remembers her day": a few dated recent thoughts ride into context and she can pick
one up, instead of starting cold.

**Acceptance criteria:**
- [ ] The dated last-24h block in `build_system_prompt` (rolling window, `LUMI_THOUGHTS_WINDOW_H`, per-user filtered, each entry timed, hard cap).
- [ ] Thoughts older than the window are excluded; an empty window → **no block**.
- [ ] A recurring thought softly colors the v0.6 mood (tone, never competence).
- [ ] Unit tests (fixed clock + temp repo) — the block carries only last-24h dated entries, capped; window boundary respected; A's thought never appears for B. No paid calls.

### LUMI-050 — Proactive nudge — silent `%think` on the timer + graduate-to-spoken

**Description:**
Generalize the v0.4 idle **nudge**: instead of always speaking a fixed opener, it mostly fires
`%think` **silently** (record only) and **occasionally graduates** a thought to a spoken turn.
(`/core`, `/tui` glue.)

**What needs to be done:**
- Reuse `should_nudge(last_activity, now, interval, quiet_hours)` as the trigger; add a
  **per-session cap** (paced — "periodical" means occasional, not every few minutes).
- On fire: run `%think` (LUMI-047) → record. **Silent** by default (nothing in chat).
- **Graduate-to-spoken:** a configurable fraction (a ratio / a "strength" threshold) deliver the
  thought through the **existing hidden self-turn** (the thought becomes the hidden self-prompt; her
  reply is shown + saved as a normal message). Mark the `Thought.spoken=true`.
- Config: `LUMI_THOUGHTS` (on/off), interval, quiet hours, per-session cap, spoken ratio.

**Dependencies:** LUMI-047 (the `%think` engine).

**Expected result:**
After idle time (paced), Лілі thinks on her own — mostly silently; once in a while she says one
aloud, conversationally.

**Acceptance criteria:**
- [ ] Timer-driven `%think` (reusing `should_nudge` + a per-session cap + quiet hours), **silent** by default.
- [ ] A configurable fraction **graduate** to a spoken turn via the hidden self-turn; `spoken` is set.
- [ ] All paths **record** to the diary; silent fires print nothing.
- [ ] Unit tests (fixed clock, mock model) — fires at the boundary, honors the cap/quiet hours, the silent/spoken split honors the ratio. No real sleeps, no paid calls.

### LUMI-051 — Input router (`/`·`%`·chat) + `%`-grammar + access gate

**Description:**
Route the input line on its first character and add the manual directive grammar — so you can type
`%think[!] [about] {topic}` — plus the **silent-vs-shared** access gate. (`/tui`, with a thin
`/core` firing seam.)

**What needs to be done:**
- **Router:** `/command` → a command; `%directive` → fire a mental act; else → a normal chat
  message. An unknown/malformed `%name` **degrades to plain chat** (never an error). A typed `%…` is
  **consumed by the router** (never saved as a user message, like `/…`).
- **Grammar:** `%<name>[!] [connector] [topic]` — `!` (glued) = **print** (open), no `!` = silent;
  the **connector** (`about`/`про`/`на тему`/`щодо`/`:`) is **optional**, stripped if present;
  `topic` = optional free text that may contain **`{placeholders}`** (LUMI-048).
- **Three surfacings:** **silent** (record only) / **open** (the raw thought shown as her inner
  voice, a `💭`-styled line, **not** persisted as a message) / **spoken** (graduates to a turn).
  Manual `%think!` → open.
- **Access gate (silent-vs-shared, not blanket owner-only):** anyone may invite a **surfaced**
  thought (open/spoken, never silent) — **rate-limited**, **isolation-respecting**, her **agency**
  honored; **silent firing** and the **raw cross-user stream** are **owner-only**. Configurable; in
  v0 (single owner) all forms are available.

**Dependencies:** LUMI-047 (the engine to fire), LUMI-048 (`{placeholders}` in the topic).

**Expected result:**
Typing `%think`, `%think!`, `%think about {last_thought}`, `%wonder! us` fires the directive with
the right mode/seed; the chat input stays clean; non-owners can only invite *surfaced* thoughts.

**Acceptance criteria:**
- [ ] Router: `/` command, `%` directive, else chat; unknown `%name` → plain chat; `%…` never saved as a message.
- [ ] Grammar parse: `!` mode, optional connector stripped (EN + UK), optional topic with `{placeholders}` resolved.
- [ ] Three surfacings — silent / open (`💭`, not a message) / spoken; `%think!` → open.
- [ ] Access gate: a non-owner can't fire **silent** nor read the **raw** stream (gets a surfaced/filtered path); rate-limited.
- [ ] Unit tests (mock model) for the parser + the gate. No paid calls.

### LUMI-052 — `/thoughts` command + `thoughts_show` policy

**Description:**
A read command in the `/mood` `/closeness` `/inner` family — shows the recent dated diary — plus the
show/log/memory policy. (`/tui`.)

**What needs to be done:**
- `/thoughts` — render the recent thought-stream, **grouped/stamped by time** (the dated diary).
  For the **owner**: the full (per-config window) stream; for a **non-owner** (v1.3+): a
  **per-user-filtered** view ("thoughts from our talks") via `thoughts_for` (LUMI-046) — never the
  whole cross-user stream.
- `thoughts_show` modes: **hidden** (default — the raw stream isn't shown to the chatting user) /
  **admin-visible** / **off**.
- **Log** the stream (the v0.3 logged tier), and **never** write the raw thoughts to long-term
  memory (only the digested v0.16 impression persists; not pinned here).

**Dependencies:** LUMI-046 (the store).

**Expected result:**
`/thoughts` shows her recent dated thoughts; the raw stream stays out of long-term memory and (by
default) out of the chatting user's view.

**Acceptance criteria:**
- [ ] `/thoughts` renders the recent dated diary; owner sees the window, a non-owner sees a per-user-filtered view.
- [ ] `thoughts_show` modes (hidden default / admin / off).
- [ ] The raw stream is **logged**, **never** persisted to long-term memory.
- [ ] Unit tests (fixed clock + temp repo) — `/thoughts` renders dated entries; the non-owner view is filtered. No paid calls.
