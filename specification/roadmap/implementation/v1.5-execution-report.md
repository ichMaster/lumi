## Detailed Results

### LUMI-192: S1 — async POST (74f644e)
`Core` gains a single ordered worker queue: `reply()` returns the instant `validate()` passes; the
post-turn tail (closeness + 2×append + recall index, exact order kept) is one queued job. Drains at
the next prompt-build / `end_session` / atexit; a failed job retries once on the next drain.
`LUMI_ASYNC_POST` (default off → byte-identical; the worker never starts). 9 tests.

### LUMI-193: S2 — SQLite message journal (9bfe204)
`state/sqlite_store.py SqliteRepository(JsonRepository)`: messages + closeness in `<store>.db` (WAL,
JSON-per-row, indexed) — O(1) INSERT/UPSERT per turn; light aggregates stay in `store.json`. First
open adopts an existing JSON store (idempotent). `LUMI_STORE_BACKEND=json|sqlite` (default json).
21 contract tests run the storage contract against BOTH backends.

### LUMI-194: S2b — vectors on SQLite (8e4fe10)
A `vectors` table (PK user_id+msg_id, kind-indexed), float32 LE `BLOB` (~4× smaller); per-search
SELECT — the ~450 MB JSONL is never RAM-resident; one-time streamed adoption without re-embedding.
Same VectorStore contract + float32-ranking tolerance pins. 11 more contract tests.

### LUMI-195: integration + migration + docs (626d902)
Full-stack integration (async+sqlite: returns before persist, ordered landing, O(1) store.json,
abort recovery, inert defaults); `scripts/migrate_store.py` (backup → migrate, idempotent,
`--export-json` rollback); `.env.example` + LATENCY.md marked shipped. 4 tests.
