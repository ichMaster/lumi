- Layer 2 per-step routing (LUMI-158, gated OFF): behind LUMI_TOOL_STEP_ROUTING the Anthropic tool-loop digs continuation rounds on a cheap step tier and speaks the final round on the call's model (the R2 two-pass); the per-round cache log tags each round's actual model; the OpenAI/Gemini loops are untouched.
- Fix: /model haiku no longer 400s — thinking/effort are now gated per model (Haiku 4.5 supports neither; its think box stays empty by design).
- Cost observability (LUMI-159): closed as operator-run — the tier vars are live and the per-kind cache/cost reports accumulate for a later measured note.

Version 0.36.0 (2026-06-22)
---------------------------
v0.36 — Lean memory III: facts core + fact recall. Facts now reach the prompt three ways (mirroring messages) — a static identity-core, a per-turn auto fact-RAG push, and a recall pull tool — plus a hygiene path:
- LUMI-141 — Fact embedding + recall(scope=facts): VectorRecord gains a `kind` discriminator ("message"|"fact"); each LongTermFact is embedded as a kind="fact" vector on write + a one-off backfill (content-addressed, idempotent). search_vectors gains a kind filter; recall / recall_moments / the recall tool gain scope = messages | facts | all (default messages = byte-identical; facts never leak into the message paths). Per-user isolated.
- LUMI-142 — The core (identity-core) flag + lifecycle: LongTermFact gains an additive `core` flag (persisted). Curated three ways — a one-off backfill (first run picks the core from all facts), an initial guess at extraction (the [C] marker), and a session-start re-flag that re-ranks the core=true pool to LUMI_FACTS_CORE_MAX with boundaries/agreements pinned. The re-flag's input is the small core pool (cost-neutral). Off when the cap is 0.
- LUMI-143 — Shrink the facts block (core static): with LUMI_FACTS_CORE_ONLY on, the prompt injects only the core facts instead of the digest (safe fallback when none flagged); the session-start re-flag replaces the per-turn facts-digest call. LUMI_RECALL_SCOPE sets the recall tool's default scope. A dropped tail fact stays reachable via recall(scope=facts).
- LUMI-144 — Auto fact-RAG (per-turn push): a `# Релевантні факти` block in the volatile tail injects the top-K relevant non-core facts each turn (LUMI_FACTS_RAG / _K), deduped against the core block; trusted, per-user, degrades to no block.
- LUMI-145 — Facts hygiene: an additive `obsolete` flag, filtered out of every fact path (core block, auto fact-RAG, recall) but kept in the store (non-destructive, reversible). A /review-facts Claude Code skill drives propose -> human review -> apply (duplicate / outdated / irrelevant); never auto-obsoletes a core fact.
Additive contract changes (VectorRecord.kind, LongTermFact.core/obsolete) — old records load with the defaults; ARCHITECTURE + contract tests updated. Off by default throughout; byte-identical when off. 1156 tests green.

Version 0.30.0 (2026-06-20)
---------------------------
- Semantic recall III — chunking long messages (v0.30, LUMI-116..119): a refinement of the v0.16 index + v0.17 context-expansion, not a new capability. A message over LUMI_RAG_CHUNK_THRESHOLD is split into ~chunk_chars passages (paragraph/sentence boundaries, small overlap), each embedded as its OWN vector; a short message stays one chunk (= v0.16). "Search fine, show coarse" — now at two levels.
- The chunker (core/chunking.py): pure, deterministic chunk_text(text, *, chunk_chars, overlap, threshold) — order-preserving, full coverage, never raises; degenerate input (empty / one huge token) handled. New LUMI_RAG_CHUNK* config, off by default.
- VectorRecord gains parent_msg_id + chunk_index (additive, back-compatible: a one-chunk message keeps the message id and chunk_index 0). The chunk's msg_id is content-addressed over parent|index|text (idempotent re-chunking). The vectors-staleness tag appends @chunk<chars> so a chunk-param change re-embeds the history. Index-on-write + backfill emit one record per chunk when on, one per message when off. (Contract: ARCHITECTURE §Semantic recall + the memory-records contract test updated with the field change.)
- Two-level expansion: search ranks per chunk; each hit renders as its PASSAGE (matched chunk ± chunk_w adjacent chunks of the same message, anchor marked) inside the ±rag_w message thread — the whole long message is never re-injected. /recall reuses the same assembly; off → byte-identical to v0.17.
- /recall current-session filter: /recall now skips the active conversation by default (the current chat is already on-screen) so a session's own echoes don't bury older sources; `/recall !all` (alias !here) includes it. core.recall/recall_moments gain exclude_session.
- Isolation extended over chunks: search, the chunk window, and the message window all run only over the requesting user's data (A<->B contract). Embedder mocked — no paid calls. Full suite 942 -> 972.
