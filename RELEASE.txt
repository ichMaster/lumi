Lumi — Release notes
====================

Version 0.5.0 (2026-06-07)
---------------------------
- v0.5 — Emoji rendering (LUMI-023…024): Лілі's emotion shows as an emoji next to her reply (Лілі 😄✨:). A renderer swap over the locked v0.3 channel — no contract change. v0 is now a complete terminal companion.
- Editable emoji map (LUMI-023): core/emoji.md (LUMI_EMOJI_PATH) maps emotion→emoji+intensity over 3 bands (repeat/accent); add/remove/change emojis without code. emoji_for + EmojiRenderer; total over the enum with a built-in default fallback.
- Emoji next to the reply in the TUI (LUMI-024): each reply labelled with her emotion glyph; intensity emphasis; the status line keeps the text readout.
- Emojis in her text too: Лілі weaves a sparing emoji or two into her replies (canon, always on).

Version 0.4.0 (2026-06-07)
---------------------------
- v0.4 — Ambient context & idle nudge (LUMI-019…022): Лілі knows when/where she is and can break a long silence first.
- Time-legible prompts (LUMI-019): every message carries its date-time; recalled short summaries are dated.
- WorldContext provider (LUMI-020): injected clock for time/calendar + config-gated weather (Open-Meteo), news (RSS), location; graceful when a source is off or fails.
- Ambient block (LUMI-021): the "now/here" block is injected into the system prompt (colors tone, never competence), fetched at startup and on /new.
- Idle nudge (LUMI-022): after a configurable silence Лілі speaks first via a hidden self-started turn from core/nudges.md; random opener, off by default, rate-limited, quiet-hours-aware.
- Richer weather: current + feels-like + today's high/low + the day's forecast; LUMI_WEATHER_URL configurable.
- TUI: always show the current style in the status line (incl. 'normal').
- Fix: strip a leading [timestamp] the model may echo from the timestamped history.

Version 0.3.0 (2026-06-07)
---------------------------
- v0.3 — Emotion field (LUMI-013…018): every reply is now a validated EmotionState
  {reply, emotion, intensity}. The fixed 9-value enum + EmotionState contract and the
  IEmotionRenderer interface are locked; Core.reply returns EmotionState (was str) and
  persists the field on assistant messages.
- Structured output: the model emits its state via a set_state tool (enum + 0–1
  constrained); the core validates/repairs it (unknown emotion -> calm, intensity
  clamped, missing reply -> surfaced error) and logs every repair.
- Reliable emotion with extended thinking on: an inline <emotion>name intensity</emotion>
  tag (parsed + stripped like <think>), reconstructed in replayed history so it doesn't
  drift away over a long conversation; the set_state tool wins when forceable (thinking off).
- LogRenderer (the v0.3 "logged" tier) writes the field per turn; the TUI shows the
  current emotion + intensity in the status line, plus an always-visible thinking:on/off
  indicator.
- Tests: 166 (mock model — no paid APIs); ruff clean.

Version 0.2.1 (2026-06-07)
---------------------------
- In-session compaction: older messages of a long session fold into a running
  per-session digest (floating verbatim window 40 + batch 20) instead of being
  dropped; end-of-session summary length now scales with session size.
- Answer styles (/style): 16 base styles (Ukrainian, by category) + 6 Лілі-voiced
  meta-style presets; combinable, per-session, injected at the end of the prompt
  as a prioritized directive with concrete length limits.
- Thinking: reasoning is wrapped in <think>…</think> and parsed out of the reply,
  shown in a separate Thinking box (last turn only) — it never leaks into the chat
  or the store.
- TUI polish: type-ahead with send-on-your-turn gating, 3-line input, macOS
  clipboard via pbcopy, borderless chat box for clean copy, /new (clears + saves),
  /prompt, and quit that summarizes the session first.
- Interface in English (footer, hints, statuses, notifications); Лілі's name and
  her replies stay Ukrainian.
- ./lumi launcher script for the TUI.
- Docs: docs/MEMORY.md and docs/STYLES.md implementation references; README refresh.
- Tests: 123 (mock model — no paid APIs); ruff clean.

Version 0.2.0 (2026-06-07)
---------------------------
- v0.2 — Three-layer memory (LUMI-007…012): the core and Repository are now
  user-scoped (user_id on every record) with a data-level isolation invariant;
  rolling-window session history; end-of-session ShortSummary; accumulated
  LongTermFact; startup rehydration of the user's context into the prompt;
  TUI /memory and /forget commands.
- Extended thinking (config-switchable): Anthropic adaptive thinking on Opus 4.8
  via .env (LUMI_THINKING on/off, LUMI_EFFORT low|medium|high|xhigh|max); model
  selectable via LUMI_MODEL.
- TUI: Лілі's reasoning shown greyed above her reply; a technical status line
  (online/requesting…/offline) plus a separate statistics line (tokens +
  latency + running totals); quit no longer blocks on slow housekeeping calls.
- Roadmap: Mood of the day moved to v0.5 (after emoji); v0.6–v0.8 renumbered.
- Tests: 85 (mock model — no paid APIs); ruff clean.

Version 0.1.0 (2026-06-06)
---------------------------
- v0.1 — Skeleton and canon (the foundation): interface-independent core with a
  single reply(user_text, session) contract; no interface logic leaks into it.
- LLMClient seam over Claude Haiku (Anthropic SDK), with a MockLLMClient and a
  bounded retry — tests never call paid APIs.
- Лілі's authored Ukrainian canon loaded verbatim as the system prompt.
- Repository interface + local JSON store; session messages persist and reload.
- Textual TUI: scrollable history, input, clean exit; model calls run off the UI
  thread; failures degrade to a readable line.
- TUI polish: color-coded speakers, spaced/bordered panels, Markdown rendering of
  Лілі's replies, clipboard copy (Ctrl+Y / Ctrl+O), clear screen (Ctrl+L), and a
  mouse-selection toggle (Ctrl+T).
- Tooling: uv project, ruff + pytest (41 tests, mock model — no paid APIs).
