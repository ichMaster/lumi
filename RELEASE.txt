Lumi — Release notes
====================

Version 0.9.0 (2026-06-09)
---------------------------
- v0.9 — Richer short memory (LUMI-034…036): each finished session is summarized in ONE call into two tiers — a detailed summary plus a one-line gist (ShortSummary gains `gist`; «СТИСЛО:» line, with a first-sentence fallback; pre-v0.9 records migrate to ""). New Repository.summaries_since(user, since_date) window query (per-user isolated). The prompt's short-memory block becomes two tiers: the last 5 conversations in detail + the recent days at a glance.
- Days at a glance → per-day digests (DaySummary{user_id, date, summary, count, ts}, a new `day_summaries` store section): instead of injecting every per-session gist (which ballooned with many short sessions/day), each day is consolidated from its gists into ONE cohesive ≤4-sentence day summary. Regenerated LAZILY at prompt-build time and COUNT-BASED — a day refreshes only when its session count changes (today as it accrues; a past day only when it gains sessions). Best-effort: a consolidation error never blocks the turn.
- Prompt order: the day digests ("Памʼять про розмови в останні дні") come FIRST, then the last-5 detailed ("Памʼять про останні розмови"). No raw per-session gists are injected — the gist is only the input to the day consolidation.
- Long-term facts (LongTermFact) untouched; per-user isolation holds throughout. Updated the ShortSummary + DaySummary contract tests and ARCHITECTURE §Memory. Mocked-model tests only (no paid calls); 300 tests green.

Version 0.8.0 (2026-06-08)
---------------------------
- v0.8 — Biorhythms merged into the daily mood (LUMI-031…033): three COMPUTED sine cycles — physical (23d) / emotional (28d) / intellectual (33d) — from her natal birth date, with high/low/rising/falling/critical labels. Deterministic (unlike the model-written horoscope), unit-tested. Fed into the v0.6 mood call so the daily reading + resolution blend horoscope + biorhythms. Cached once per local day; colors tone/energy, never competence. /biorhythm command; LUMI_BIORHYTHMS toggle (on by default).
- Hormonal (menstrual) cycle: a fourth, PHASED body rhythm — менструація → фолікулярна → овуляція → лютеїнова → ПМС — computed from an authored anchor ("Цикл: день 1 — DD.MM.YYYY, довжина NN" in core/natal.md) + the injected clock. Merged into the SAME daily mood under a shared "integrate these body rhythms … reflect in the RESOLUTION" directive; colors tone/energy/sensitivity, never competence. Part of her embodied self-image, not a medical claim. Shown in /biorhythm; LUMI_CYCLE toggle (on by default).
- Mood: the biorhythm/cycle merge is an explicit integration directive (weave into the reading together with the transits, reconcile conflicts, reflect in the resolution) — not a separate appended block.
- Fix: strip a leading [YYYY-MM-DD …] date-stamp from a reply even when the model drops the closing "]" (kept normal text that merely starts with "[" or a number).
- Roadmap: moved the face wardrobe to after closeness (v0.9 → v0.11); generated/uploaded/executed the v0.8 issues.

Version 0.7.2 (2026-06-08)
---------------------------
- TUI send/receive sound: a short blip on send (Tink) + receive (Glass). Backend chain — macOS afplay + system sounds (primary; this pygame build has no mixer module), pygame.mixer synth fallback, else silent. OFF by default (LUMI_SOUND); Ctrl+S toggles at runtime; the status line shows sound:on/off. Never sounds the idle nudge; degrades to a no-op with no audio device.
- Auto-style: Лілі chooses her own answer style each turn — the whole palette rides in the system prompt, she picks the fitting style (preferring meta/"mega"-styles), writes in it, and declares it as <style>name</style> (parsed + stripped, mirror of the emotion tag).
- /style is now a recommendation, not a switch: /style <name> sets a soft per-session bias she still decides on; /style auto clears it. The status bar shows her chosen style and who picked it — (Лілі), or (ти) when she followed your recommendation.
- Docs/spec: ARCHITECTURE §Answer styles + docs/STYLES.md refreshed for the auto-style model; style tests rewritten.

Version 0.7.1 (2026-06-08)
---------------------------
- Viewer idle-relax: the face eases to calm after LUMI_FACE_IDLE_SECONDS of an unchanged signal (default 120s; 0 = off); the next emotion wakes it. Honors EMOTION.md's ttl_ms for the static tier. Injected clock — tested without a display.
- Timestamped face signal: the core writes "<emotion> <intensity> <YYYY-MM-DD HH:MM:SS>" each turn, so every turn's line is unique; the viewer detects new turns by the full line (idle-relax wakes even on a repeated emotion). PNGs decoded via Pillow (this pygame build lacks PNG support).
- TUI input lock: the input box is disabled while Лілі replies (a turn or the save-on-exit) and re-enabled + refocused when it's your turn — no more typing into a box that won't send.
- Idle nudges: user-editable openers in core/nudges.md; the nudge-file test is now content-agnostic (asserts loading behaviour, not specific lines).
- Roadmap (planning only, no code): specced upcoming v0 phases — biorhythms merged into the mood (v0.8), face wardrobe (v0.9), richer short memory (v0.10), closeness (v0.11), inner life (v0.12-13), emotional memory (v0.14-15), semantic recall / RAG (v0.16-17).

Version 0.6.1 (2026-06-08)
---------------------------
- clock: system_clock now returns LOCAL time, not UTC — the mood "day" (and per-message timestamps + the ambient "now") were keyed on UTC, so just after local midnight they were a day behind.
- mood: the full daily reading is now persisted to .lumi/mood.log (it previously went only to a logger with no configured sink, so it was effectively unsaved).
- mood: the resolution is honest/objective — not artificially positive (hard, low, tense days too) — and describes her STATE only, with no advice or recommendations.
- mood: require the detailed reading body before the resolution (the model sometimes emitted only «РЕЗОЛЮЦІЯ:»), and fix split_resolution to handle an inline marker.
- docs: add docs/MOOD.md — a worked example of how the mood prompt is assembled.

Version 0.6.0 (2026-06-08)
---------------------------
- v0.6 — Mood of the day (LUMI-025…027): Лілі has a model-generated daily temperament that colors her tone + the emotions she leans toward, never her competence. An experiment for daily variation, not an astrological claim.
- Daily mood engine (LUMI-025): once per local day a mood call writes a horoscope-flavored reading from her natal chart (core/natal.md, skyfield-verified) + today's date; the full reading is logged, only a short resolution is held; cached per local day, recomputed at local midnight; on by default (LUMI_MOOD), graceful on failure. No astronomy engine — a real-ephemeris test confirmed the model can't compute precise transits, and variation (not precision) is the goal.
- Prominent resolution block (LUMI-026): only the resolution is injected into the system prompt as a prioritized block (like the v0.5 style header), biasing tone + emitted emotion, never competence; the full reading never enters the prompt.
- /mood command (LUMI-027): shows the day's resolution on demand.

Version 0.3.0 (2026-06-07)
---------------------------
- v0.3 — Emotion field (LUMI-013…018): every reply is now a validated EmotionState
  {reply, emotion, intensity}. The fixed 9-value enum + EmotionState contract and the
  IEmotionRenderer interface are locked; Core.reply returns EmotionState (was str) and
  persists the field on assistant messages.
- Structured output: the model emits its state via a set_state tool (enum + 0–1
  constrained); the core validates/repairs it (unknown emotion -> calm, intensity
  clamped, missing reply -> surfaced error) and logs every repair.
- Reliable emotion with extended thinking on: an inline <emotion>name intensity</emotion>
  tag (parsed + stripped like <think>), reconstructed in replayed history so it doesn't
  drift away over a long conversation; the set_state tool wins when forceable (thinking off).
- LogRenderer (the v0.3 "logged" tier) writes the field per turn; the TUI shows the
  current emotion + intensity in the status line, plus an always-visible thinking:on/off
  indicator.
- Tests: 166 (mock model — no paid APIs); ruff clean.

Version 0.2.1 (2026-06-07)
---------------------------
- In-session compaction: older messages of a long session fold into a running
  per-session digest (floating verbatim window 40 + batch 20) instead of being
  dropped; end-of-session summary length now scales with session size.
- Answer styles (/style): 16 base styles (Ukrainian, by category) + 6 Лілі-voiced

  clipboard via pbcopy, borderless chat box for clean copy, /new (clears + saves),
  /prompt, and quit that summarizes the session first.
- Interface in English (footer, hints, statuses, notifications); Лілі's name and
  her replies stay Ukrainian.
- ./lumi launcher script for the TUI.
- Docs: docs/MEMORY.md and docs/STYLES.md implementation references; README refresh.
- Tests: 123 (mock model — no paid APIs); ruff clean.

Version 0.2.0 (2026-06-07)

  rolling-window session history; end-of-session ShortSummary; accumulated

- Tests: 85 (mock model — no paid APIs); ruff clean.

Version 0.1.0 (2026-06-06)
---------------------------

  single reply(user_text, session) contract; no interface logic leaks into it.
- LLMClient seam over Claude Haiku (Anthropic SDK), with a MockLLMClient and a
  bounded retry — tests never call paid APIs.
- Лілі's authored Ukrainian canon loaded verbatim as the system prompt.
- Repository interface + local JSON store; session messages persist and reload.
- Textual TUI: scrollable history, input, clean exit; model calls run off the UI
  thread; failures degrade to a readable line.
- TUI polish: color-coded speakers, spaced/bordered panels, Markdown rendering of
  Лілі's replies, clipboard copy (Ctrl+Y / Ctrl+O), clear screen (Ctrl+L), and a
  mouse-selection toggle (Ctrl+T).
- Tooling: uv project, ruff + pytest (41 tests, mock model — no paid APIs).
